학습자가 사용 중인 회의록 도구를 확인하고 입력 방식을 결정합니다.

## 분기

| 학습자 도구 | 입력 방식 | 추가 절차 |
|---|---|---|
| **Tiro** | A. MCP 직결 | Tiro MCP가 이미 셋업되어 있는지 확인 |
| Granola / Otter / Notta / 직접 작성 | B. 붙여넣기 | 추가 셋업 없음 |
| 안 씀 | 스킵 | "회의록 도구 도입 후 다시 와주세요" 안내 |

## A. Tiro 사용자

```
mcp__claude_ai_tiro__list_notes
```
호출해서 노트 접근 가능한지 확인. 첫 노트 1건 가져와서 학습자에게 "이렇게 자비스가 회의록을 자동으로 가져올 수 있어요" 시연.

## B. 붙여넣기 사용자

학습자에게 "회의록 텍스트를 가져오는 방법"을 안내:
- 본인 도구에서 transcript 또는 요약 텍스트를 복사
- `/meeting-recap` 호출 → 자비스가 입력 대기 → 텍스트 붙여넣기

## 안내 마무리

"입력 방식 결정됐으면 `/3-2-build-meeting-recap`으로 넘어가시죠."
