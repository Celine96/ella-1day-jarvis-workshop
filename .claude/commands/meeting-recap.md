자비스는 **회의록을 만들지 않는다.** 외부 도구(Tiro / Granola / Otter / Notta 등)의 출력물을 입력으로 받는다.

## 입력 방식

- **A. MCP 직결 (Tiro 사용자)**: 인자에 노트 ID 또는 검색어 → `mcp__claude_ai_tiro__get_note_transcript`
- **B. 붙여넣기 (다른 도구)**: 회의록 텍스트를 인자 또는 후속 입력으로

## 1. 회의록 파싱

- 회의 제목·일시·참석자·주요 토픽 추출
- 결정사항 / 액션아이템 / 미해결 질문 분리

## 2. 액션아이템 아이젠하워 분류

각 액션아이템에 대해 **담당자**와 **사분면** 동시 판단:

| 사분면 | 분류 기준 |
|---|---|
| **Q1 — Do** | 요한님 담당 + 마감 7일 이내 |
| **Q2 — Plan** | 요한님 담당 + 마감 8일 이상 또는 미정 |
| **Q3 — Delegate** | 타인 담당 — 위임 담당자 추천 |
| **Q4 — Delete** | 폐기 가능한 항목 |

Q3 위임 항목은 **담당자만 추천**한다. 위임 메일·메시지 발송은 학습자가 직접 처리한다. (자동 발송·초안 작성 X)

## 3. 출력 형식

```
[자비스] {{회의명}} 회의록 정리

📅 일시: ...
👥 참석자: ...

## 결정사항
• ...

## 액션아이템 (아이젠하워 분류)

🔴 Q1 — 요한님 본인이 7일 이내
• [항목] — 마감 [날짜]

🟠 Q2 — 요한님 본인이 시간 확보 필요
• [항목] — 마감 [날짜]

🟡 Q3 — 위임 (담당자 추천)
• [항목] → 담당 추천: [이름]

⚪ Q4 — 스킵
• [항목]

## 미해결 질문
• ...
```

## 원칙

- 회의록을 자비스가 만들지 않는다 — 외부 도구 결과를 입력으로 받는다
- Q3는 담당자 추천만 — 자비스가 위임 메일을 자동 작성·발송하지 않는다
- 위임 추천 시 담당자가 명확하지 않으면 "담당 미정 — 요한님이 직접 처리 필요" 명시

## 다음 단계 안내 (워크숍 첫 실행 시 한정)

워크숍 진행 중 첫 실행이라면 결과 출력 후 다음 안내를 추가합니다:

> "회의록 정리 결과를 검토하시고, Q1 본인 마감과 미해결 질문 답변 일정을 확인해주세요. Q3 위임 메일은 본인이 직접 작성·발송하시면 됩니다. 끝나면 `/3-3-connect-option-channel`로 옵션 채널 연결 단계로 가시면 됩니다."
