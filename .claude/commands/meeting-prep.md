description: 다음 회의 참석자·맥락·논의점 + 회의 안건의 아이젠하워 분류
---

인자: `[회의명 또는 시간]` (선택). 인자 없으면 가장 가까운 다음 회의 자동 선택.

## 1. 회의 식별

- Calendar에서 다음 회의 1건 조회 (인자 있으면 매칭, 없으면 최근접)
- 회의 제목·시간·참석자·장소·설명 추출

## 2. 컨텍스트 수집

다음 소스 병렬 조회:
1. **Gmail 검색** — 회의 제목 + 참석자 이메일로 최근 30일 이내 관련 스레드
2. **Calendar 과거 이력** — 같은 회의명·참석자의 이전 미팅
3. (옵션) 연결된 회의록 도구 (Tiro 등) — 같은 참석자와의 이전 회의록

## 3. 안건 우선순위화 (아이젠하워)

회의 설명·관련 메일에서 안건 추출 → 4사분면 분류:

| 사분면 | 회의 안건 분류 기준 |
|---|---|
| **Q1 — Do** | 이 자리에서 결정·처리해야 하는 안건 |
| **Q2 — Schedule** | 정보 공유·논의 (가져갈 가치 있음) |
| **Q3 — Delegate** | 다른 자리에서 다뤄도 되는 안건 |
| **Q4 — Delete** | 빠져도 되는 항목 (시간 절약) |

## 4. 출력 형식

```
[자비스] {{회의명}} 미팅 준비

📅 시간: HH:MM~HH:MM (Y분)
👥 참석자: 이름 (역할), ...
📍 장소: ...

## 컨텍스트
[지난 미팅 요약 또는 관련 메일 핵심 1~3줄]

## 안건 (아이젠하워 분류)

🔴 Q1 — 이 자리에서 결정
• [안건] — [관련 배경 1줄]

🟠 Q2 — 논의·정보공유
• [안건]

🟡 Q3 — 위임/다른 자리
• [안건] → [어디서 처리]

⚪ Q4 — 스킵 가능
• [안건]

## 요한님이(가) 미리 준비할 것
• [bulleted prep items]
```

## 원칙

- 미팅 시작 30분~2시간 전에 호출 가정
- 안건이 명확하지 않으면 "안건 미정 — 호스트에게 확인 필요" 명시
- 참석자 중 처음 만나는 사람이면 "처음 만남" 태그
