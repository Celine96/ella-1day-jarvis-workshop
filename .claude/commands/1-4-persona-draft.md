## 1단계 — 1-3 인터뷰 답변 반영 (즉시)

1-3 `/1-3-inventory-import`에서 메모리에 저장된 답변을 페르소나 파일에 자동 반영:
- **§2 회사 컨텍스트** ← 1-3 답변 1
- 페르소나 §1에 "연결 도구·옵션 채널 후보" 한 줄 ← 1-3 답변 2·3
- **§4 Q1 자동 분류 규칙** ← 1-3 답변 4 (3개)
- **§4 Q4 자동 분류 규칙** ← 1-3 답변 5

1-3 답변이 메모리에 없으면 → 1-3 인터뷰부터 진행하도록 안내.

## 2단계 — 추가 인터뷰 (톤·보고 형식·시간 블록)

### A. 톤 선택 (1분)
> "요한님, 제 톤은 어떻게 가져갈까요? 다음 중 골라주세요.
>
> A. **정중·효율** — '요한님, 다음과 같이 정리했습니다.'
> B. **친근·간결** — '오늘 하실 일이에요.'
> C. **격식** — '보고드립니다.'
>
> 또는 본인 톤을 자유롭게 한 줄로 알려주셔도 됩니다."

→ **§3 톤** 자동 채움

### B. 보고 형식 (1분)
> "매일 받으실 메일은 어떤 형태가 편하세요?
>
> - **사분면 헤더 + 불릿** (가장 많이 쓰임, 추천)
> - 산문
> - 표
> - 혼합"

→ **§5 보고 형식** 자동 채움

### C. 시간 블록 정책 점검 (2분)
> "기본값은 다음과 같아요:
> - 오전 09~12 = Q1·Q2 집중 작업
> - 오후 13~17 = 회의·커뮤니케이션
> - 회의 사이 30분 버퍼
>
> 이대로 OK이신가요? 본인 라이프스타일에 맞게 바꾸실까요?
> 예) '저는 새벽이 집중 시간이라 06~09를 Q1·Q2로', '점심 13~14는 절대 안 옮김'."

→ **§6 시간 블록 정책** 자동 채움

## 3단계 — 페르소나 파일 자동 저장

모든 답변을 받으면:
1. `.claude/agents/{{JARVIS_NAME}}.md` 파일을 Read
2. 해당 섹션을 Edit 도구로 자동 채움
3. Save

학습자에게 출력:
```
✅ {{JARVIS_NAME}}의 페르소나가 완성됐습니다.

생성된 페르소나 파일: .claude/agents/{{JARVIS_NAME}}.md
- §1 정체성 ✓
- §2 회사 컨텍스트 ✓
- §3 톤 ✓
- §4 Q1·Q4 분류 규칙 ✓
- §5 보고 형식 ✓
- §6 시간 블록 정책 ✓
- §7 발송 채널 정책 (기본값) ✓

다음 Part에서 본인 Gmail/Calendar에 연결한 뒤,
이 페르소나로 작동하는 첫 운영 커맨드를 만들어볼게요.
```

## 안전장치

- 학습자 답변이 비어 있는 항목은 `[학습자 보강 필요]`로 표기 — 후속 챕터에서 자비스가 짚어줌
- 자비스가 직접 추측해서 채우지 말 것 (학습자 의도와 어긋남 방지)
- 파일 덮어쓰기 전에 학습자에게 미리보기 제시
- 민감 정보(비밀번호·계좌·계약 금액 등) 입력하지 말도록 안내 (페르소나 §7 보안 정책)
