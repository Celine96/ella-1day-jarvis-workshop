인자: `[기간]` (선택). 미지정 시 기본값: 최근 7일 미답 메일 (최대 50통).

## 1. 메일 수집

- `mcp__claude_ai_Gmail__search_threads`로 `is:unread newer_than:7d -label:Promotions -label:Social`
- 각 스레드 메타데이터: 발신자·제목·미리보기·시간

## 2. 아이젠하워 매트릭스 분류

페르소나 §4 규칙 우선 적용:

| 사분면 | 메일 분류 기준 |
|---|---|
| **Q1 — Do** | 답장 필수 + 24h 이내 — 페르소나 Q1 규칙 / 외부 거래처 / 직속 임원 |
| **Q2 — Schedule** | 답장 필요 + 시간 확보 — 전략적 검토 요청 / 의사결정 위임 |
| **Q3 — Delegate** | 위임 가능 — 운영 문의 / 다른 팀 관할 → 위임 대상 추천 |
| **Q4 — Delete** | 무시·삭제 — 뉴스레터 / 콜드 영업 / 자동 알림 |

## 3. Q1 답장 초안 일괄 작성

페르소나 톤 적용해 답장 초안 생성:
- 수신자·제목·본문 작성
- **Gmail 임시보관함에 저장** (자동 발송 절대 금지)
- 본문 길이: 페르소나 §5 보고 형식 선호 따름

## 4. 출력 형식

```
[{{JARVIS_NAME}}] 받은편지함 정리 (총 N건)

🔴 Q1 — Do (답장 필수, 답장 초안 작성됨)
• [발신자] [제목 일부]
  → 임시보관함 저장 ✓

🟠 Q2 — Schedule (시간 확보 후 답장)
• [발신자] [제목] — 추천 처리 시간: [언제]

🟡 Q3 — Delegate (위임 추천)
• [발신자] [제목] → 담당 추천: [이름·팀]

⚪ Q4 — Delete (무시·삭제 가능)
• [발신자] [제목] (사유: 뉴스레터/자동 알림/콜드 영업)
• ... (총 N건)

## 요약
- Q1 N건 답장 초안 → 임시보관함 (검토 후 발송)
- Q3 N건 위임 추천
- Q4 N건 무시 가능 — 결정 비용 사라짐
```

## 원칙

- 모든 메일에 사분면 라벨 부착 (누락 없이)
- Q1 답장 초안은 페르소나 톤 100% 적용
- 자동 발송 절대 금지 — 임시보관함 저장만
- Q4는 명시적으로 "무시 가능" 표기 (요한님이 안심하고 무시할 수 있도록)
