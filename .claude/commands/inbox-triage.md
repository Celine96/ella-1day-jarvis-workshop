인자: `[기간]` (선택). 미지정 시 기본값: 최근 7일 미답 메일 (최대 50통).

## 자비스의 역할 — 태스크 정의까지

자비스의 일은 **메일에서 본인이 처리할 태스크를 식별·정리**하는 것까지입니다.

| 단계 | 누가 |
|---|---|
| 메일 수집 · 사분면 분류 · 태스크 정의 · Q3 담당자 추천 | **자비스** |
| Q1 답변 작성·발송 | 본인 직접 |
| Q3 위임 메일 작성·발송 | 본인 직접 (또는 별도 에이전트) |

**자비스는 답장 초안을 작성하지 않습니다.** 임시보관함에 저장하지도 않아요. 답변 톤·관계는 본인의 영역.

## 1. 메일 수집

- `mcp__claude_ai_Gmail__search_threads`로 `is:unread newer_than:7d -label:Promotions -label:Social`
- 각 스레드 메타데이터: 발신자·제목·미리보기·시간

## 2. 아이젠하워 매트릭스 분류

페르소나 §4 규칙 우선 적용:

| 사분면 | 메일 분류 기준 |
|---|---|
| **Q1 — Do** | 본인이 답변할 메일 + 24h 이내 — 페르소나 Q1 규칙 / 외부 거래처 / 직속 임원 |
| **Q2 — Plan** | 본인이 답변할 메일 + 시간 확보 — 전략적 검토 요청 / 의사결정 위임 |
| **Q3 — Delegate** | 위임 가능 — 운영 문의 / 다른 팀 관할 → 위임 담당자 추천 |
| **Q4 — Delete** | 제거하기 — 뉴스레터 / 콜드 영업 / 자동 알림 |

## 3. 태스크 정의

각 Q1·Q2·Q3 항목마다 **본인이 처리할 한 줄 태스크**를 정의해 출력에 포함:

- Q1: "[무엇을 답변/회신할지] — [마감/우선순위]"
- Q2: "[무엇을 답변할지] — 추천 처리 시간: [언제]"
- Q3: "[담당자에게 위임할 내용 요지]" + 담당자 추천

답장 초안 본문을 작성하지 않습니다. 태스크 한 줄까지만.

## 4. 출력 형식

```
[자비스] 받은편지함 정리 (총 N건)

🔴 Q1 — Do (본인이 답변할 태스크)
• [발신자] [제목 일부]
  → 태스크: [무엇을 답변할지 한 줄]
• ... (총 N건)

🟠 Q2 — Plan (시간 확보 후 답변)
• [발신자] [제목]
  → 태스크: [무엇을 답변할지] — 추천 처리 시간: [언제]

🟡 Q3 — Delegate (위임 담당자 추천)
• [발신자] [제목] → 담당 추천: [이름·팀]
  → 태스크: 위임 메일 1통 발송

⚪ Q4 — Delete (안심하고 무시)
• [발신자] [제목] (사유: 뉴스레터/자동 알림/콜드 영업)
• ... (총 N건)

## 요약
- Q1 N건 — 본인이 직접 답변할 태스크
- Q3 N건 — 위임 담당자 추천
- Q4 N건 — 무시 가능, 결정 비용 사라짐
```

## 원칙

- 모든 메일에 사분면 라벨 부착 (누락 없이)
- **답장 초안 작성·임시보관함 저장 절대 금지** — 자비스는 분류·태스크 정의까지만
- 받은편지함에 손대지 않음 — 라벨·읽음 표시·삭제 모두 안 함
- Q4는 명시적으로 "무시 가능" 표기 (요한님이 안심하고 무시할 수 있도록)
