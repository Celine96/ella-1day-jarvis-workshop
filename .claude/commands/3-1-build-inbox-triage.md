`.claude/commands/inbox-triage.md`를 학습자와 함께 검토하고 첫 실행으로 안내합니다.

## 흐름

1. 자비스의 역할 경계부터 명확히
   - 자비스 = 메일에서 **태스크 식별·정리** (분류 + 한 줄 태스크 정의 + Q3 담당자 추천)
   - 본인 = Q1 답변 작성·발송, Q3 위임 메일 작성·발송
   - **자비스는 답장 초안을 작성하지 않습니다.** 임시보관함에도 손대지 않아요.
2. 파일 §1~§4 짧게 설명
   - §1 메일 수집 (최근 7일 미답)
   - §2 아이젠하워 분류
   - §3 태스크 정의 — Q1·Q2·Q3 각 항목에 한 줄 태스크 정의
   - §4 출력 형식
3. 학습자에게 첫 실행 안내:
   ```
   /inbox-triage
   ```
4. 결과 검토 — Q1 본인 답변 태스크 5건, Q3 위임 추천 6건, Q4 무시 28건
5. Q4 항목의 의미 강조 — "이 N건은 안심하고 무시하셔도 됩니다"

## 자주 나오는 튜닝

- "Q4가 너무 많아요" → 페르소나 §4 Q4 규칙 다듬기 ("이 발신자는 항상 Q4")
- "Q1 태스크 정의가 모호해요" → 페르소나 §5 보고 형식 다듬기, "한 줄 태스크는 동사로 시작"
- "Q3 위임 추천이 부정확" → 페르소나 §1에 팀 구성·역할 한 줄 추가

## 결정 비용 사라지는 순간

학습자에게 "지금 미답 메일을 10분 안에 처리할 수 있는 상태가 됐어요. 무엇에 답변해야 할지, 누구에게 위임해야 할지, 무시해도 되는 게 무엇인지 — 자비스가 정리했어요. 답변 작성 결정은 본인이 직접." 안내.

## 다음 단계 안내

`/inbox-triage` 첫 실행 + 결과 검토가 끝나면 다음 안내를 출력합니다:

> "받은편지함 정리 완료. 이제 회의록을 자비스에 연결할 차례예요. `/3-2-connect-meeting-tool`을 입력해주세요."
