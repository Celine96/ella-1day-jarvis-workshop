`.claude/commands/meeting-recap.md`를 학습자와 함께 검토하고 첫 실행으로 안내합니다.

## 흐름

1. 파일 §1~§4 짧게 설명
   - 자비스는 회의록을 만들지 않음 — 외부 도구 결과를 입력으로 받음
   - 액션아이템을 아이젠하워로 분류 (Q1·Q2 본인 / Q3 위임 / Q4 폐기)
   - Q3 항목은 후속 메일 초안 자동 작성 → 임시보관함
2. 첫 실행:
   - Tiro 사용자: `/meeting-recap "[회의명]"` 으로 자동 가져오기
   - 다른 도구 사용자: `/meeting-recap` 후 회의록 텍스트 붙여넣기
3. 결과 검토 + 임시보관함의 후속 메일 초안 확인 안내

## 자주 나오는 튜닝

- "후속 메일 톤이 어색" → 페르소나 §3 톤 다듬기
- "위임 대상 추천이 부정확" → 페르소나 §1에 팀 구성·외부 자문 명단 추가
- "결정사항과 액션아이템 구분이 모호" → 페르소나 §1에 "결정사항 = 회의에서 합의된 것 / 액션아이템 = 누군가가 그 후 실행할 것" 명시

## 안전 강조

후속 메일은 **임시보관함에만** 저장. 자비스가 자동 발송하지 않음. 검토 후 학습자가 직접 "보내기" 클릭.

## 다음 단계 안내

`/meeting-recap` 첫 실행 + 후속 메일 검토가 끝나면 다음 안내를 출력합니다:

> "회의록 처리 완료. 이제 본인 옵션 채널 1개를 자비스에 추가할게요. `/3-3-connect-option-channel`을 입력해주세요."
