`.claude/commands/meeting-prep.md`를 열어서 학습자와 함께 검토합니다.

## 흐름

1. 파일 §1~§4 짧게 설명
   - §1 회의 식별 (인자 매칭 또는 최근접 자동)
   - §2 컨텍스트 수집 (Gmail + Calendar 과거 이력 + 회의록 도구)
   - §3 안건 우선순위 (아이젠하워)
   - §4 출력 형식
2. 학습자에게 첫 실행 안내:
   ```
   /meeting-prep
   ```
   또는 특정 회의:
   ```
   /meeting-prep "[회의명]"
   ```
3. 결과 검토 후 튜닝 (페르소나 §4 Q1/Q4 규칙 다듬기)

## 자주 나오는 튜닝 예시

- "참석자 정보가 너무 적어요" → 페르소나 §1에 "참석자별 직전 상호작용 요약 추가" 명시
- "안건이 안 추출돼요" → 회의 설명·관련 메일이 부족. "안건 미정" 표기 그대로 활용 권장

## 다음 단계 안내

`/meeting-prep` 첫 실행 + 튜닝이 끝나면 다음 안내를 출력합니다:

> "회의 준비 커맨드 작동 확인. 이제 한 주 단위(`/weekly-review`)로도 시야를 넓혀볼게요. `/2-4-build-weekly-review`를 입력해주세요."
