# 3-1. 메일·회의 자비스에 위임

::: tip Step 9 / 11 · Part 3
이 챕터에서 — **본인 일의 매트릭스 Q3(위임) 사분면을 자비스에 맡깁니다.** 받은편지함과 회의록 두 영역에서 자비스가 본인이 처리할 태스크를 식별·정리하고, Q3 항목은 위임 담당자까지 추천해드려요. 답변·발송 결정은 항상 본인이 직접.
:::

## Q3 위임 사분면 — 자비스의 핵심 영역

아이젠하워 매트릭스의 **Q3(위임)** 사분면은 "비중요 + 긴급"이에요. 본인이 직접 안 해도 누군가에게 넘기면 처리되는 항목들. 매일 본인 시간을 가장 비효율적으로 잡아먹는 영역이기도 해요 — **"이건 누구에게 넘기지?"** 결정 비용이 매번 들거든요.

Part 3의 핵심은 **이 Q3 사분면 전체를 자비스에 맡기는 것**이에요. 메일이든 회의록이든 일정이든 — 본인 일 중 Q3에 해당하는 모든 항목을 자비스가 식별·추천해드립니다. 위임 결정의 인지 비용을 자비스가 가져가는 셈이에요.

이 챕터에서는 두 영역의 Q3를 자비스에 맡깁니다.

| 영역 | 자비스가 식별·추천하는 Q3 | 운영 커맨드 |
|---|---|---|
| **받은편지함** | 위임 가능 메일 + 담당 부서·팀 추천 | `/inbox-triage` |
| **회의록** | 위임 가능 액션아이템 + 담당자 추천 | `/meeting-recap` |

다음 챕터(3-2)에서 자동 routine으로 등록하면 매일 본인이 호출 안 해도 자비스가 Q3를 자동 정리해드려요.

## Gmail 쓰기 권한 추가

자비스가 다음 챕터(3-2)에서 등록할 **자동 routine 산출물**을 매일 본인 메일함으로 자기발송하려면 쓰기 권한이 필요해요. 2-2에서 받은 읽기 권한 위에 **"메일 보내기"** 한 가지를 추가합니다.

| 시점 | Gmail | Calendar |
|---|---|---|
| 이전 (2-2) | 읽기 | 읽기 |
| **이 챕터** | 읽기 + 쓰기 | 읽기 유지 |

::: warning 쓰기 권한 = 본인 메일함 자기발송만
쓰기 권한을 추가해도 자비스가 외부로 메일을 자동 발송하는 일은 **절대 일어나지 않습니다.** 페르소나 §7에 박힌 경계 — 자비스가 쓰기 권한으로 할 수 있는 건 한 가지뿐이에요.

- 자동 routine 산출물(매일 아침 사분면 정리)을 본인 메일함으로 자기발송 — 외부 발송 아님

**답장 작성·발송도 자비스가 하지 않습니다.** 자비스의 일은 **메일에서 태스크를 식별·정리**하는 것까지. 답변 톤·발송 결정은 항상 본인이 직접 — 이 분리가 자비스를 안심하고 쓸 수 있게 만드는 핵심입니다.
:::

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-1-add-write-permission
```

자비스가 권한 추가 절차를 안내합니다.

1. Google 계정 보안 페이지에서 클로드 코드 권한 항목 열기
2. OAuth 동의 화면에서 **"메일 보내기"** 체크 추가
3. 클로드 코드에서 Gmail MCP 재인증

검증:

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-2-mcp-test
```

이전 출력에 더해 **"Gmail 쓰기 권한 활성"** 한 줄이 추가되어 있으면 준비 완료예요.

## 메일 → 태스크 정의 — /inbox-triage

받은편지함을 자비스가 정리하는 단계예요. 자비스의 일은 단순합니다 — **메일에서 본인이 처리할 태스크를 식별하고 사분면으로 정리**해드립니다. Q1은 본인이 답변할 태스크, Q3는 위임할 담당자 추천, Q4는 안심하고 무시.

::: tip 자비스는 답장을 쓰지 않습니다
자비스가 메일에서 하는 일은 **태스크 식별·정리까지**예요. 답장 작성·발송은 자비스가 하지 않습니다.

| 단계 | 누가 |
|---|---|
| **메일 → 태스크 정의·사분면 분류 (Q3 위임 담당자 추천 포함)** | **자비스의 일** |
| Q1 답변 작성·발송 | 본인이 직접 |
| Q3 위임 메일 작성·발송 | 본인이 직접 (또는 별도 에이전트 — 부록 §F) |

답장은 본인의 톤·관계가 담기는 산출물이라 자비스가 일방적으로 작성할 영역이 아니에요. 자비스가 잘하는 건 **"어떤 메일에 답변해야 하는가"** 를 한눈에 보여드리는 것까지.
:::

먼저 작동 설명서를 살펴봅니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-1-build-inbox-triage
```

자비스가 `.claude/commands/inbox-triage.md` 핵심 동작 3가지를 보여줍니다.

1. **미답 메일 50통 수집** — 최근 7일 받은편지함
2. **페르소나 Q1·Q4 규칙 우선 적용해 4사분면 분류** — Q3 메일은 위임 담당자 추천
3. **본인 처리 태스크 리스트로 정리** — Q1·Q3 항목별로 "무엇을 해야 하는지" 한 줄 태스크 정의

받은편지함 자체에는 손대지 않아요. 메일 삭제·라벨 변경·읽음 표시·답장 초안 작성 모두 안 일어납니다. 자비스는 분류·태스크 정의 결과만 출력합니다.

이제 본인 받은편지함을 자비스가 정리할 차례.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/inbox-triage
```

자비스가 다음과 같이 출력합니다.

```
[자비스] 받은편지함 정리 (총 47건)

🔴 Q1 — Do (본인이 답변할 태스크)
• [발신자] [제목 일부]
  → 태스크: IR 후속 자료 회신 (8:30 안)
• ... (예: 5건)

🟠 Q2 — Plan (시간 확보 후 답변)
• [발신자] [제목]
  → 태스크: PoC 성공 지표 정의 답신 — 추천 처리 시간: 내일 오전
• ... (예: 8건)

🟡 Q3 — Delegate (위임 담당자 추천)
• [발신자] [제목] → 담당 추천: [팀]
  → 태스크: 위임 메일 1통 발송
• ... (예: 6건)

⚪ Q4 — Delete (안심하고 무시)
• [발신자] [제목] (사유: 뉴스레터)
• ... (예: 28건)

## 요약
- Q1 5건 — 본인이 직접 답변할 태스크
- Q3 6건 — 위임 담당자 추천
- Q4 28건 — 무시 가능, 결정 비용 사라짐
```

본인이 봐야 할 건 Q1·Q3 두 사분면뿐. 47건이 11건으로 줄어든 셈이에요.

### Q1 — 본인이 답변할 태스크

자비스가 Q1으로 분류한 메일은 본인이 직접 답변하셔야 합니다. 자비스의 가치는 답장 작성이 아니라 **"이 5건이 오늘 본인이 답변해야 할 메일"** 이라는 것을 한눈에 보여드리는 데 있어요.

각 Q1 항목에 자비스가 한 줄로 정리해준 **태스크 정의**를 보고 본인이 답변 작성·발송하시면 됩니다.

### Q4 무시의 자유

`/inbox-triage`의 진짜 가치는 사실 Q1·Q3가 아니라 **Q4**에 있어요.

> **"이 28건은 안심하고 무시하셔도 됩니다."**

페르소나 §4 Q4 자동 분류 규칙(뉴스레터·콜드 영업·자동 알림)에 매칭된 메일들 — 본인이 일일이 열어 "답장해야 하나?" 결정 비용을 안 들여도 돼요. **47건을 11건(Q1·Q3)으로 줄이는 게 자비스의 첫 큰 시간 절약**입니다.

### Q3 위임 메일 처리

자비스가 추천한 Q3 위임 담당 부서·팀을 보고 위임 메일을 작성·발송하시면 돼요. **위임 메일 발송은 본인이 직접** (자비스가 자동 발송하지 않음).

| 옵션 | 위임 방식 | 시점 |
|---|---|---|
| **A. 본인 직접 위임** | 자비스 추천을 보고 본인이 위임 메일 작성·발송 | 워크숍 안 흐름 (지금) |
| **B. 에이전트 자동화** (가능·제안) | 별도 에이전트가 위임 메일 초안 자동 작성·임시보관함 저장 | 워크숍 후 자가 개발 |

**B 옵션 — 에이전트 자동화**는 워크숍 후 부록 [§F. 에이전트 확장](/appendix#f-에이전트-확장) 가이드로 자가 개발하실 수 있어요. 자비스의 인간 우선 원칙(자비스 자체는 답장 작성·외부 발송을 하지 않음)을 보존하면서 위임 워크플로를 한 단계 더 자동화합니다.

## 회의록 Q3 위임 — /meeting-recap

회의록의 Q3 위임 영역(타인 담당 액션아이템)을 자비스가 맡는 단계예요. 회의가 끝나면 결정사항·액션아이템·미해결 질문이 흩어진 상태인데, **자비스가 사분면으로 정리하고 Q3 액션아이템은 담당자까지 추천**해드립니다.

### 자비스는 회의록을 만들지 않습니다

회의록 자체를 만드는 일 — 음성을 녹음하고, 발화를 transcript로 옮기고, 요약을 정리 — 은 이미 시장에 좋은 도구가 많아요 (Tiro · Granola · Otter · Notta). 자비스의 가치는 그 다음 단계예요.

| 단계 | 누가 |
|---|---|
| 회의 녹음·전사·요약 | 본인이 이미 쓰는 도구 |
| **회의록 → 액션아이템 사분면 분류 (특히 Q3 위임 담당자 추천)** | **자비스의 일** |

### 본인 회의록 도구 확인

먼저 본인이 어떤 도구로 회의록을 만드는지 자비스에게 알려주세요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-connect-meeting-tool
```

자비스가 도구별로 입력 방식을 안내합니다.

| 본인 도구 | 입력 방식 |
|---|---|
| **Tiro** | A. MCP 직결 (자동) |
| Granola · Otter · Notta · 기타 | B. 붙여넣기 (transcript 복사) |
| 직접 작성 (메모장 · Notion 등) | B. 붙여넣기 |
| 회의록 안 씀 | 이 섹션 스킵 가능 — 회의록 도구 도입 후 다시 |

### /meeting-recap 작동 설명서

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-build-meeting-recap
```

자비스가 핵심 동작 2가지를 보여줍니다.

1. **회의록 파싱** — 제목·일시·참석자·결정사항·액션아이템·미해결 질문 자동 추출
2. **액션아이템 아이젠하워 분류** — 담당자·마감일 기준 사분면 배치
   - Q1: 본인 담당 + 7일 이내
   - Q2: 본인 담당 + 8일 이상
   - Q3: 타인 담당 — **위임 담당자 추천**
   - Q4: 폐기 가능

### Q3 위임 액션아이템 처리

자비스가 Q3 액션아이템마다 **담당자를 추천**해드립니다. 위임 메일·메시지 발송은 메일 Q3와 같은 두 옵션이에요.

| 옵션 | 위임 방식 | 시점 |
|---|---|---|
| **A. 본인 직접 위임** | 자비스 추천 담당자에게 본인이 위임 메일·메시지 작성·발송 | 워크숍 안 흐름 (지금) |
| **B. 에이전트 자동화** (가능·제안) | 별도 에이전트가 위임 메일 초안 자동 작성·임시보관함 저장 | 워크숍 후 자가 개발 |

자가 자동화를 원하시면 부록 [§F. 에이전트 확장](/appendix#f-에이전트-확장) 참고.

- Gmail 쓰기 권한 추가 (OAuth) — 다음 챕터 routine 자기발송용
- `/inbox-triage` 첫 실행 — 받은편지함 47건 → Q1 5건(본인 답변) · Q3 6건(위임) · Q4 28건(무시)
- `/meeting-recap` 첫 실행 — 회의록 액션아이템 사분면 분류 + Q3 위임 담당자 추천

자비스의 일은 항상 **태스크 식별·정리**까지. 답변·발송·위임 결정은 본인이 직접. 본인이 매일 들이던 "이 메일 답변해야 하나?" "이 액션아이템 누가 가져가지?" 결정 비용이 자비스에 맡겨진 셈이에요. 다음 챕터에서는 이 정리를 **자동 routine**으로 등록해, 본인이 매일 호출 안 해도 자비스가 알아서 정리해두는 상태로 만듭니다.
:::
