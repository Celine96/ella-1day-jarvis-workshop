# 3-1. 메일·회의 자비스에 맡기기

::: tip Step 9 / 11 · Part 3
이 챕터에서 — **본인 일의 매트릭스 Q3(위임) 사분면을 자비스에 맡깁니다.** 받은편지함과 회의록 두 영역에서 자비스가 위임 가능 항목을 식별하고 담당자까지 추천해드려요.
:::

## Q3 위임 사분면 — 자비스의 핵심 영역

아이젠하워 매트릭스의 **Q3(위임)** 사분면은 "비중요 + 긴급"이에요. 본인이 직접 안 해도 누군가에게 넘기면 처리되는 항목들. 매일 본인 시간을 가장 비효율적으로 잡아먹는 영역이기도 해요 — **"이건 누구에게 넘기지?"** 결정 비용이 매번 들거든요.

Part 3의 핵심은 **이 Q3 사분면 전체를 자비스에 맡기는 것**이에요. 메일이든 회의록이든 일정이든 — 본인 일 중 Q3에 해당하는 모든 항목을 자비스가 식별·추천해드립니다. 위임 결정의 인지 비용을 자비스가 가져가는 셈이에요.

이 챕터에서는 두 영역의 Q3를 자비스에 맡깁니다.

| 영역 | 자비스가 식별·추천하는 Q3 | 운영 커맨드 |
|---|---|---|
| **받은편지함** | 위임 가능 메일 + 담당 부서·팀 추천 | `/inbox-triage` |
| **회의록** | 위임 가능 액션아이템 + 담당자 추천 | `/meeting-recap` |

다음 챕터(3-2)에서 자동 routine으로 등록하면 매일 본인이 호출 안 해도 자비스가 Q3를 자동 정리해드려요.

## Gmail 쓰기 권한 추가

자비스가 Q3 메일의 답장 초안을 Gmail 임시보관함에 저장하려면 쓰기 권한이 필요해요. 2-2에서 받은 읽기 권한 위에 **"메일 보내기"** 한 가지를 추가합니다.

| 시점 | Gmail | Calendar |
|---|---|---|
| 이전 (2-2) | 읽기 | 읽기 |
| **이 챕터** | 읽기 + 쓰기 | 읽기 유지 |

::: warning 쓰기 권한 = 발송이 아니다
쓰기 권한을 추가해도 자비스가 외부로 메일을 자동 발송하는 일은 **절대 일어나지 않습니다.** 페르소나 §7에 박힌 경계 — 자비스가 할 수 있는 건 두 가지뿐이에요.

- 답장 초안을 Gmail 임시보관함에 저장 (본인이 검토 후 직접 발송)
- 자동 routine 산출물을 본인 메일함으로 자기발송 (외부에 안 나감)

자비스의 분류·작성 속도는 빠르게, 외부 발송 결정은 본인이 직접 — 이 분리가 자비스를 안심하고 쓸 수 있게 만드는 핵심입니다.
:::

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-1-add-write-permission
```

자비스가 권한 추가 절차를 안내합니다.

1. Google 계정 보안 페이지에서 클로드 코드 권한 항목 열기
2. OAuth 동의 화면에서 **"메일 보내기"** 체크 추가
3. 클로드 코드에서 Gmail MCP 재인증

검증:

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-2-mcp-test
```

이전 출력에 더해 **"Gmail 쓰기 권한 활성"** 한 줄이 추가되어 있으면 준비 완료예요.

## 메일 Q3 위임 — /inbox-triage

받은편지함의 Q3 위임 영역을 자비스가 맡는 단계예요. 자비스가 미답 메일을 사분면으로 분류하고, **Q3 메일은 담당 부서·팀까지 추천**해드립니다. 본인이 안 해도 되는 메일이 한눈에 정리되는 셈이에요.

먼저 작동 설명서를 살펴봅니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-1-build-inbox-triage
```

자비스가 `.claude/commands/inbox-triage.md` 핵심 동작 3가지를 보여줍니다.

1. **미답 메일 50통 수집** — 최근 7일 받은편지함
2. **페르소나 Q1·Q4 규칙 우선 적용해 4사분면 분류** — Q3 메일은 위임 담당자 추천
3. **Q1 항목에 답장 초안 자동 작성** → Gmail 임시보관함 저장 (외부로 안 나감)

이제 본인 받은편지함을 자비스가 정리할 차례.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/inbox-triage
```

자비스가 다음과 같이 출력합니다.

```
[자비스] 받은편지함 정리 (총 47건)

🔴 Q1 — Do (답장 필수, 답장 초안 작성됨)
• [발신자] [제목 일부]
  → 임시보관함 저장 ✓
• ... (예: 5건)

🟠 Q2 — Plan (시간 확보 후 답장)
• [발신자] [제목] — 추천 처리 시간: 내일 오전
• ... (예: 8건)

🟡 Q3 — Delegate (위임 추천)
• [발신자] [제목] → 담당 추천: [팀]
• ... (예: 6건)

⚪ Q4 — Delete (제거하기 가능)
• [발신자] [제목] (사유: 뉴스레터)
• ... (예: 28건)

## 요약
- Q1 5건 답장 초안 → 임시보관함 (검토 후 발송)
- Q3 6건 위임 추천
- Q4 28건 무시 가능 — 결정 비용 사라짐
```

본인 Gmail에서 일어난 변화는 두 가지뿐이에요.

- **임시보관함에 답장 초안 5건 저장** (Q1 항목)
- **외부 발송 0건** — 자비스가 외부로 보낸 메일은 없음

받은편지함 자체에는 변화가 없습니다. 메일 삭제·라벨 변경·읽음 표시 모두 안 일어나요.

### 임시보관함 검토

Gmail 좌측 메뉴에서 **"임시보관함(Drafts)"** 을 열어 자비스가 작성한 답장 초안 5건을 확인하세요. 페르소나 톤·원본 인용·보고 형식이 모두 반영된 상태입니다.

초안을 검토하고 마음에 들면 **"보내기"** 버튼을 누르면 메일이 나갑니다. 마음에 안 들면 자비스에게 그대로 말씀하시면 됩니다 — 예: "투자자 김OO 메일 답장은 좀 더 정중하게 다시 써주세요" — 자비스가 임시보관함의 초안을 다시 작성합니다.

### Q4 무시의 자유

`/inbox-triage`의 진짜 가치는 사실 Q1·Q3가 아니라 **Q4**에 있어요.

> **"이 28건은 안심하고 무시하셔도 됩니다."**

페르소나 §4 Q4 자동 분류 규칙(뉴스레터·콜드 영업·자동 알림)에 매칭된 메일들 — 본인이 일일이 열어 "답장해야 하나?" 결정 비용을 안 들여도 돼요. **47건을 11건(Q1·Q3)으로 줄이는 게 자비스의 첫 큰 시간 절약**입니다.

### Q3 위임 메일 처리

자비스가 추천한 Q3 위임 담당 부서·팀을 보고 위임 메일을 작성·발송하시면 돼요. **위임 메일 발송은 본인이 직접** (자비스가 자동 발송하지 않음).

| 옵션 | 위임 방식 | 시점 |
|---|---|---|
| **A. 본인 직접 위임** | 자비스 추천을 보고 본인이 위임 메일 작성·발송 | 워크숍 안 흐름 (지금) |
| **B. 에이전트 자동화** (가능·제안) | 별도 에이전트가 위임 메일 초안 자동 작성·임시보관함 저장 | 워크숍 후 자가 개발 |

**B 옵션 — 에이전트 자동화**는 워크숍 후 부록 [§F. 에이전트 확장](/appendix#f-에이전트-확장) 가이드로 자가 개발하실 수 있어요. 자비스의 인간 우선 원칙을 보존하면서 위임 워크플로를 한 단계 더 자동화합니다.

## 회의록 Q3 위임 — /meeting-recap

회의록의 Q3 위임 영역(타인 담당 액션아이템)을 자비스가 맡는 단계예요. 회의가 끝나면 결정사항·액션아이템·미해결 질문이 흩어진 상태인데, **자비스가 사분면으로 정리하고 Q3 액션아이템은 담당자까지 추천**해드립니다.

### 자비스는 회의록을 만들지 않습니다

회의록 자체를 만드는 일 — 음성을 녹음하고, 발화를 transcript로 옮기고, 요약을 정리 — 은 이미 시장에 좋은 도구가 많아요 (Tiro · Granola · Otter · Notta). 자비스의 가치는 그 다음 단계예요.

| 단계 | 누가 |
|---|---|
| 회의 녹음·전사·요약 | 본인이 이미 쓰는 도구 |
| **회의록 → 액션아이템 사분면 분류 (특히 Q3 위임 담당자 추천)** | **자비스의 일** |

### 본인 회의록 도구 확인

먼저 본인이 어떤 도구로 회의록을 만드는지 자비스에게 알려주세요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-connect-meeting-tool
```

자비스가 도구별로 입력 방식을 안내합니다.

| 본인 도구 | 입력 방식 |
|---|---|
| **Tiro** | A. MCP 직결 (자동) |
| Granola · Otter · Notta · 기타 | B. 붙여넣기 (transcript 복사) |
| 직접 작성 (메모장 · Notion 등) | B. 붙여넣기 |
| 회의록 안 씀 | 이 섹션 스킵 가능 — 회의록 도구 도입 후 다시 |

### /meeting-recap 작동 설명서

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-build-meeting-recap
```

자비스가 핵심 동작 2가지를 보여줍니다.

1. **회의록 파싱** — 제목·일시·참석자·결정사항·액션아이템·미해결 질문 자동 추출
2. **액션아이템 아이젠하워 분류** — 담당자·마감일 기준 사분면 배치
   - Q1: 본인 담당 + 7일 이내
   - Q2: 본인 담당 + 8일 이상
   - Q3: 타인 담당 — **위임 담당자 추천**
   - Q4: 폐기 가능

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

### Q3 위임 액션아이템 처리

자비스가 Q3 액션아이템마다 **담당자를 추천**해드립니다. 위임 메일·메시지 발송은 메일 Q3와 같은 두 옵션이에요.

| 옵션 | 위임 방식 | 시점 |
|---|---|---|
| **A. 본인 직접 위임** | 자비스 추천 담당자에게 본인이 위임 메일·메시지 작성·발송 | 워크숍 안 흐름 (지금) |
| **B. 에이전트 자동화** (가능·제안) | 별도 에이전트가 위임 메일 초안 자동 작성·임시보관함 저장 | 워크숍 후 자가 개발 |

자가 자동화를 원하시면 부록 [§F. 에이전트 확장](/appendix#f-에이전트-확장) 참고.
