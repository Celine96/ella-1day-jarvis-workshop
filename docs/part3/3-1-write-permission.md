# 3-1. 쓰기 권한 + /inbox-triage

::: tip Step 9 / 13 · Part 3
이 챕터에서 — **자비스에게 Gmail 쓰기 권한을 추가하고, 받은편지함 미답 메일을 사분면 정리부터 Q1 답장 초안 임시보관함 저장까지 함께 끝냅니다.**
:::

## 왜 쓰기 권한이 필요한가

2-2에서 자비스에게 Gmail · Calendar 읽기 권한만 줬어요. 자비스가 신뢰 가는 결과를 보여주기 전에 외부로 무엇이 나가지 않게 하는 안전장치였습니다. Part 2를 거치면서 자비스의 분류와 정리가 본인 직관과 맞다는 걸 검증했으니, 이제 한 단계 권한을 올릴 차례예요.

| 시점 | Gmail | Calendar |
|---|---|---|
| 이전 (2-2) | 읽기 | 읽기 |
| **이 챕터 (3-1)** | 읽기 + 쓰기 | 읽기 유지 |

쓰기 권한을 추가하는 이유는 두 가지입니다.

- **`/inbox-triage`의 답장 초안 저장** — 자비스가 Q1 메일에 답장 초안을 작성해 Gmail 임시보관함에 넣으려면 쓰기 권한이 필요해요.
- **자동 routine의 자기발송** — 3-4에서 매일 아침 8시·일요일 저녁 8시에 자비스가 본인 메일함으로 브리핑을 자동 발송하는 routine을 등록합니다. 본인 → 본인 발송이라도 메일을 만들고 보내는 동작이라 쓰기 권한이 필요합니다.

이 두 동작은 모두 **외부로 메일이 나가지 않는 범위 안**에서 일어나요. 임시보관함은 본인이 "보내기"를 누르기 전엔 외부에 안 나가고, 자기발송은 본인 메일함 안에서만 움직입니다.

::: warning 쓰기 권한 = 발송이 아니다
쓰기 권한을 추가해도 자비스가 외부로 메일을 자동 발송하는 일은 **절대 일어나지 않습니다.** 이 경계는 페르소나 §7에 박혀 있고, 운영 커맨드 작동 설명서에도 명시되어 있어요.

자비스가 쓰기 권한으로 할 수 있는 것은 두 가지뿐입니다.

- 답장 초안을 Gmail 임시보관함에 저장 (본인이 검토 후 직접 발송)
- routine 산출물을 본인 메일함으로 자기발송 (외부에 나가지 않음)

자비스가 외부 수신자에게 보낼 메일을 자동 발송하는 동작은 — 답장이든 신규 메일이든 — 페르소나 §7에서 명시적으로 차단되어 있어요. 자비스의 분류·작성 속도는 빠르게, 외부 발송 결정은 본인이 직접 — 이 분리가 자비스를 안심하고 쓸 수 있게 만드는 핵심입니다.
:::

## Gmail 쓰기 권한 추가

2-2에서 거쳤던 OAuth 흐름과 똑같아요. 차이는 이번엔 "메일 읽기" 위에 **"메일 보내기"** 권한 한 가지를 추가로 켜는 것뿐입니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-1-add-write-permission
```

자비스가 권한 추가 절차를 안내합니다. 흐름은 다음과 같아요.

1. Google 계정 보안 페이지에서 클로드 코드 권한 항목 열기
2. OAuth 동의 화면에서 **"메일 보내기"** 체크 추가 (이전 "메일 읽기"는 그대로 유지)
3. 클로드 코드에서 Gmail MCP 재인증

3단계까지 끝나면 자비스의 OAuth "출입증"에 쓰기 권한이 한 줄 더 추가된 상태가 됩니다. 다른 권한(메일 삭제·라벨 변경 등)은 굳이 켜지 마세요.

검증은 2-2와 같은 방식으로요.

**[커맨드 입력]**

```
/2-2-mcp-test
```

이전 출력에 더해 **"Gmail 쓰기 권한 활성"** 한 줄이 추가되어 있어야 합니다. 이 한 줄이 보이면 자비스에게 답장 초안을 맡길 준비가 끝난 거예요.

## /inbox-triage 작동 설명서 확인

`/inbox-triage`는 받은편지함 미답 메일을 한꺼번에 정리하는 운영 커맨드예요. 2-3·2-4와 같은 패턴으로, 실행 전에 작동 설명서를 한 번 살펴보고 갑니다.

**[커맨드 입력]**

```
/3-1-build-inbox-triage
```

자비스가 `.claude/commands/inbox-triage.md`를 열어 핵심 동작 3가지를 보여줍니다.

1. **미답 메일 50통 수집** — 최근 7일 받은편지함에서 본인이 답장하지 않은 메일을 모음
2. **페르소나 Q1·Q4 규칙 우선 적용해 4사분면 분류** — 페르소나에 박힌 분류 규칙이 1순위
3. **Q1 항목에 답장 초안 자동 작성** → Gmail 임시보관함 저장 (외부로 나가지 않음)

작동 설명서 형식은 `/morning-brief`와 같은 구조예요. 마음에 안 드는 부분이 있으면 자비스에게 그대로 말씀하시면 자동 수정됩니다.

## 첫 실행

이제 본인 받은편지함을 자비스가 정리할 차례입니다.

**[커맨드 입력]**

```
/inbox-triage
```

자비스가 받은편지함을 한 번에 훑고 다음과 같이 출력합니다.

```
[자비스] 받은편지함 정리 (총 47건)

🔴 Q1 — Do (답장 필수, 답장 초안 작성됨)
• [발신자] [제목 일부]
  → 임시보관함 저장 ✓
• ... (예: 5건)

🟠 Q2 — Plan (시간 확보 후 답장)
• [발신자] [제목] — 추천 처리 시간: 내일 오전
• ... (예: 8건)

🟡 Q3 — Delegate (위임 추천)
• [발신자] [제목] → 담당 추천: [팀]
• ... (예: 6건)

⚪ Q4 — Delete (제거하기 가능)
• [발신자] [제목] (사유: 뉴스레터)
• ... (예: 28건)

## 요약
- Q1 5건 답장 초안 → 임시보관함 (검토 후 발송)
- Q3 6건 위임 추천
- Q4 28건 무시 가능 — 결정 비용 사라짐
```

이 출력이 화면에 뜨는 동안, 본인 Gmail에서 일어난 일은 두 가지뿐입니다.

- **임시보관함에 답장 초안 5건 저장** (Q1 항목)
- **외부 발송 0건** (자비스가 외부로 보낸 메일은 단 한 통도 없음)

받은편지함 자체에는 변화가 없어요. 메일이 삭제되거나 라벨이 바뀌거나 읽음 표시되는 일은 없습니다. 자비스는 "보고 분류하고 초안만 만든" 거예요.

## 임시보관함 확인

Gmail 좌측 메뉴에서 **"임시보관함(Drafts)"** 을 열어보세요. 자비스가 작성한 답장 초안 5건이 들어 있을 거예요. 각 초안은 다음 특징을 갖습니다.

- **페르소나 톤**이 적용되어 있음 (1-4에서 정한 §3 톤)
- **원본 메일을 인용**해서 답장 컨텍스트가 명확함
- **본인의 보고 형식 선호**(페르소나 §5)에 맞춰 작성

초안을 하나씩 열어 검토해보세요. 마음에 들면 **"보내기"** 버튼만 누르시면 메일이 나갑니다. 마음에 안 들면 자비스에게 그대로 말씀하시면 됩니다 — 예: "투자자 김OO 메일 답장은 좀 더 정중하게 다시 써주세요" — 자비스가 임시보관함의 그 초안을 다시 작성합니다.

::: tip 검토는 본인이, 작성은 자비스가
이 분리가 쓰기 권한을 안심하고 사용하게 만드는 핵심이에요. 작성 속도는 자비스가 빠르게 처리하고, 외부 발송 결정은 항상 본인이 직접. 답장 5통 작성 시간이 30분에서 5분으로 줄어드는 셈입니다.
:::

## Q4 무시의 자유

`/inbox-triage`의 진짜 가치는 사실 Q1이 아니라 **Q4**에 있어요.

> **"이 28건은 안심하고 무시하셔도 됩니다."**

자비스가 Q4로 분류한 메일은 페르소나 §4의 Q4 자동 분류 규칙(예: 뉴스레터·콜드 영업·자동 알림)에 정확히 매칭된 메일들이에요. 본인이 일일이 열어 "답장해야 하나?" 결정 비용을 들일 필요 없는 항목입니다.

매일 받은편지함에서 30분씩 사라지던 결정 비용이 한 번에 정리됩니다. 본인이 봐야 할 건 Q1(답장)과 Q3(위임 결정) 두 사분면뿐 — 11건이에요. **47건을 11건으로 줄이는 게 자비스의 첫 큰 시간 절약이고, 그 핵심이 Q4 자동 분류**입니다.

자비스의 Q4 분류가 어색하게 느껴지는 메일이 있다면, 그건 페르소나 §4 Q4 규칙을 한 줄 더 정교하게 다듬을 단서예요. 자비스에게 그대로 말씀하시면 페르소나가 자동 업데이트됩니다.
