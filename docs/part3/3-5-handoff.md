# 3-5. 권한 회수 + 다음 단계

::: tip Step 13 / 13 · Part 3
이 챕터에서 — **워크숍 종료 후에도 자비스를 안전하게 운영하기 위한 권한 회수 가이드와 다음 단계 확장 안내까지 함께 마무리합니다.**
:::

## 마지막 한 챕터의 의미

자비스가 24/7 작동을 시작했어요. 워크숍 4시간 동안 정체성 잡기·본인 데이터 연결·자동 routine 등록까지 모두 끝났습니다. 이 마지막 챕터에서는 자비스를 안심하고 일상에서 쓸 수 있도록 다음 세 가지를 정리하고 갑니다.

- **권한 회수** — 자비스를 그만 쓰고 싶을 때 1분이면 끝나는 절차
- **자비스 손에 남은 것** — 워크숍 산출물 체크리스트
- **다음 단계** — 옵션 채널 추가·페르소나 진화·막혔을 때 도움받는 법

10분이면 충분해요.

## 권한 관리

자비스가 본인 데이터에 접근하는 모든 권한은 **언제든지 회수**할 수 있습니다. 회수 절차는 1분이면 끝나고, 회수 즉시 자비스는 본인 데이터에 더 이상 접근할 수 없게 돼요. 자비스 권한을 안심하고 켜둘 수 있는 핵심 안전망입니다.

회수가 필요해지시면 자비스에게 단계별 가이드를 받으세요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-5-handoff
```

자비스가 다음 표대로 채널별 회수 위치를 안내합니다.

| 권한 | 회수 위치 |
|---|---|
| Gmail / Calendar | Google 계정 → 보안 → 클로드 코드 액세스 → 제거 |
| Slack | Slack 워크스페이스 설정 → 앱 관리 → 클로드 코드 제거 |
| Notion | Notion 설정 → 연동 → 클로드 코드 제거 |

권한을 회수하시면 다음 routine 실행 시점에 자비스가 데이터 부족 에러를 만나며 자동으로 멈춥니다. 본인이 별도로 routine을 끄실 필요 없어요. 완전히 멈춘 뒤에 자비스를 다시 쓰고 싶으시면, 같은 OAuth 동의를 다시 거치시면 됩니다 (2-2와 같은 흐름).

## routine 일시정지 / 휴가 모드

권한 회수까지 가지 않고 잠깐 자비스를 쉬게 하는 것도 가능해요. 휴가·집중 작업 주간 같은 상황에 쓰시면 됩니다.

| 상황 | 자비스에게 한 줄로 |
|---|---|
| **이번 주 휴가** | "이번 주 routine 정지해주세요" — routine 등록 해제, 다음 주 다시 등록 |
| **시간 조정** | "평일 9시로 바꿔주세요" — cron 자동 업데이트 |
| **routine 영구 해제** | "daily routine 삭제해주세요" |

휴가에서 돌아오시면 "다시 켜주세요" 한 줄이면 routine이 같은 cron으로 다시 시작됩니다. 일시정지 동안에도 페르소나 파일·운영 커맨드는 그대로 살아 있어, 손으로 `/morning-brief`를 호출하시면 자비스가 평소대로 응답해요.

## 자비스 손에 남은 것

워크숍이 끝나는 지금, 요한님 손에 남은 자산은 다음과 같습니다.

::: tip 워크숍 산출물 체크리스트
- **자비스의 페르소나** — `.claude/agents/자비스.md` (정체성·회사 컨텍스트·톤·우선순위 규칙·시간 블록 정책)
- **운영 슬래시 커맨드 5종** — `/morning-brief` · `/meeting-prep` · `/weekly-review` · `/inbox-triage` · `/meeting-recap`
- **자동 routine 2종** — 매일 평일 08:00 · 일요일 20:00
- **Gmail · Calendar 본인 계정 MCP 연결** — 읽기 + 쓰기
- **옵션 채널 1개** — Slack · Notion · LinkedIn 중 하나
- **옵션 채널 보조 커맨드 1개** — `/slack-triage` · `/notion-okr-check` · `/linkedin-triage` 중 하나
:::

이 자산은 워크숍이 끝난 다음 날부터도 매일 같은 형태로 작동합니다. 한 번 정리해두면 본인의 일상 운영 도구가 되는 셈이에요.

## 다음 채널 확장 (선택)

이번 워크숍에서는 핵심 옵션 채널 1개만 연결했지만, 부록에는 다른 채널을 추가로 연결하는 레시피가 준비돼 있어요. 집에 가셔서 시간 되실 때 1~2개씩 추가하시면 자비스가 더 입체적인 비서가 됩니다.

- [부록 A — Slack 추가 연결](/appendix#slack-recipe) — 팀 운영·실무 중심이라면
- [부록 B — Notion 추가 연결](/appendix#notion-recipe) — 문서·OKR 중심이라면
- [부록 C — LinkedIn 추가 연결](/appendix#linkedin-recipe) — B2B 사업개발 중심이라면

추가 채널 연결 시 절차는 3-3과 같아요. 부록의 레시피를 그대로 따라가시면 30분 안에 한 채널이 더 늘어납니다.

## 페르소나 진화 (선택)

자비스를 며칠·몇 주 쓰시다 보면 분류가 본인 직관과 어긋나는 순간이 종종 옵니다. 그때마다 자비스에게 한 줄로 말씀하시면 페르소나가 즉시 다듬어져요.

```
"이 메일은 항상 Q3로 분류해주세요. 우리 회사 운영팀 관할이에요."
"이 일정은 절대 옮기면 안 됩니다. Q1 고정으로요."
"Q4 항목은 발신자만 보여주고 제목은 생략해주세요."
```

페르소나는 살아 있는 문서예요. 워크숍 끝나고 한 달 정도 이렇게 다듬으시면, **본인에게만 정확히 맞는 비서**가 됩니다. 처음 자비스 메일과 한 달 뒤 자비스 메일을 비교하시면 톤·정확도 차이가 확연히 보이실 거예요.

## Part 3 리캡

Part 3에서 자비스가 "읽고 정리하는 비서"에서 **"실제로 일하는 비서"** 로 진화했어요. 쓰기 권한 추가 → 회의록 액션아이템 정리 → 옵션 채널 1개 합류 → 자동 routine 등록까지 한 번에 묶었습니다.

| 챕터 | 한 줄 회고 |
|---|---|
| 3-1 쓰기 권한 + /inbox-triage | 받은편지함 47건 → Q1 답장 초안 5건 임시보관함 저장, Q4 무시의 자유 |
| 3-2 회의록 + /meeting-recap | 회의 끝나면 액션아이템 사분면 분류 + Q3 위임 담당자 추천 |
| 3-3 본인 옵션 채널 1개 | Slack · Notion · LinkedIn 중 핵심 1개를 자비스 운영 흐름에 합류 |
| 3-4 자동 routine 등록 | 클라우드 routine으로 매일 08:00 · 일요일 20:00 자동 발송 시작 |
| 3-5 권한 회수 + 다음 단계 | 안전 가이드 · 페르소나 진화 · 부록으로 자가 확장 |

워크숍 4시간 — 자비스 정체성(Part 1) → 자비스 깨우기(Part 2) → 자비스 24/7 작동(Part 3)의 흐름을 끝냈어요. 자비스가 24/7로 작동하기 시작했고, 매일 아침 본인 메일함에 사분면 정리 메일이 도착하는 일상이 시작됐습니다.
