# 3-3. 안심하고 위임

::: tip Step 11 / 11 · Part 3
이 챕터에서 — **워크숍 종료 후에도 자비스에 위임된 Q3 정리가 안전하게 작동하도록 권한 관리 가이드와 다음 단계 확장 안내까지 마무리합니다.**
:::

## 마지막 한 챕터의 의미

자비스가 24/7로 본인 일의 Q3를 위임받기 시작했어요. 워크숍 동안 정체성 잡기·본인 데이터 연결·Q3 위임 자동화까지 모두 끝났습니다. 이 마지막 챕터에서는 위임을 안심하고 운영할 수 있도록 다음 세 가지를 정리하고 갑니다.

- **권한 회수** — 자비스 위임을 그만 두고 싶을 때 1분이면 끝나는 절차
- **자비스 손에 남은 것** — 워크숍 산출물 체크리스트
- **다음 단계** — 옵션 채널 추가·에이전트 자가 개발·페르소나 진화

10분이면 충분해요.

## 권한 관리 — 위임 안전망

자비스가 본인 데이터에 접근하는 모든 권한은 **언제든지 회수**할 수 있습니다. 회수 절차는 1분이면 끝나고, 회수 즉시 자비스는 본인 데이터에 더 이상 접근할 수 없게 돼요. 자비스에 일을 위임하면서도 안심할 수 있는 핵심 안전망입니다.

회수가 필요해지시면 자비스에게 단계별 가이드를 받으세요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-5-handoff
```

자비스가 다음 표대로 채널별 회수 위치를 안내합니다.

| 권한 | 회수 위치 |
|---|---|
| Gmail / Calendar | Google 계정 → 보안 → 클로드 코드 액세스 → 제거 |
| Slack | Slack 워크스페이스 설정 → 앱 관리 → 클로드 코드 제거 |
| Notion | Notion 설정 → 연동 → 클로드 코드 제거 |

권한을 회수하시면 다음 routine 실행 시점에 자비스가 데이터 부족 에러를 만나며 자동으로 멈춥니다. 본인이 별도로 routine을 끄실 필요 없어요. 완전히 멈춘 뒤에 자비스를 다시 쓰고 싶으시면, 같은 OAuth 동의를 다시 거치시면 됩니다 (2-2와 같은 흐름).

## routine 일시정지 / 휴가 모드

권한 회수까지 가지 않고 잠깐 자비스를 쉬게 하는 것도 가능해요. 휴가·집중 작업 주간 같은 상황에 쓰시면 됩니다.

| 상황 | 자비스에게 한 줄로 |
|---|---|
| **이번 주 휴가** | "이번 주 routine 정지해주세요" — routine 등록 해제, 다음 주 다시 등록 |
| **시간 조정** | "평일 9시로 바꿔주세요" — cron 자동 업데이트 |
| **routine 영구 해제** | "daily routine 삭제해주세요" |

휴가에서 돌아오시면 "다시 켜주세요" 한 줄이면 routine이 같은 cron으로 다시 시작됩니다. 일시정지 동안에도 페르소나 파일·운영 커맨드는 그대로 살아 있어, 손으로 `/morning-brief`를 호출하시면 자비스가 평소대로 응답해요.

## 자비스 손에 남은 것

워크숍이 끝나는 지금, 요한님 손에 남은 자산은 다음과 같습니다.

::: tip 워크숍 산출물 체크리스트
- **자비스의 페르소나** — `.claude/agents/자비스.md` (정체성·회사 컨텍스트·톤·우선순위 규칙·시간 블록 정책·자동화 정책 §8)
- **운영 슬래시 커맨드 5종** — `/morning-brief` · `/meeting-prep` · `/weekly-review` · `/inbox-triage` · `/meeting-recap`
- **자동 routine 2종** — 매일 평일 08:00 · 일요일 20:00
- **Gmail · Calendar 본인 계정 MCP 연결** — 읽기 + 쓰기
:::

이 자산은 워크숍이 끝난 다음 날부터도 매일 같은 형태로 작동합니다. 한 번 정리해두면 본인 일의 Q3 정리가 자비스에 안정적으로 위임된 상태가 유지돼요.

## 다음 단계 (선택)

워크숍 후 본인 컨텍스트에 맞게 자비스를 확장하실 수 있어요. 세 가지 방향이 있습니다.

### 옵션 채널 추가

평소 쓰시는 Slack · Notion · LinkedIn 같은 채널을 자비스에 추가 연결하면 그 채널의 Q3 항목까지 자비스가 정리해드려요. 가이드: [부록 §C. 옵션 채널 레시피](/appendix#c-옵션-채널-레시피)

### Q3 위임 에이전트 자가 개발

위임 메일 작성·발송까지 자동화하고 싶으시면 별도 에이전트를 자가 구성하실 수 있어요. 자비스의 인간 우선 원칙을 보존하면서 위임 워크플로를 한 단계 더 자동화합니다. 가이드: [부록 §F. 에이전트 확장](/appendix#f-에이전트-확장)

### 페르소나 진화

자비스를 며칠·몇 주 쓰시다 보면 분류가 본인 직관과 어긋나는 순간이 종종 옵니다. 그때마다 자비스에게 한 줄로 말씀하시면 페르소나가 즉시 다듬어져요.

```
"이 메일은 항상 Q3로 분류해주세요. 우리 회사 운영팀 관할이에요."
"이 일정은 절대 옮기면 안 됩니다. Q1 고정으로요."
"Q4 항목은 발신자만 보여주고 제목은 생략해주세요."
```

페르소나는 살아 있는 문서예요. 워크숍 끝나고 한 달 정도 이렇게 다듬으시면, **본인에게만 정확히 맞는 비서**가 됩니다.
