# 3-4. 자동 routine 등록

::: tip Step 12 / 13 · Part 3
이 챕터에서 — **자비스를 클라우드 routine으로 등록해, 매일 아침 8시·일요일 저녁 8시에 자동 발송이 시작되는 단계까지 함께 마칩니다.**
:::

## 0번 약속의 실현

프롤로그 [0. 자비스, 들어보셨죠?](/prologue)에서 강사 자비스가 매일 아침 보내드린 사분면 정리 메일을 보여드렸죠. **그게 30초 후 요한님 메일함에서 일어납니다.** 이 챕터 끝에서 즉시 발송 테스트를 한 번 돌리면, 본인 자비스가 보낸 첫 사분면 메일이 본인 메일함에 떨어지게 돼요.

워크숍 4시간의 결산 단계예요. Part 1에서 페르소나를 잡았고, Part 2에서 본인 데이터에 연결했고, Part 3 앞 챕터에서 쓰기 권한·회의록·옵션 채널까지 갖췄어요. 이번 챕터에서 그 모든 능력을 자동 routine으로 묶어, 자비스가 본인 노트북이 꺼져 있는 동안에도 작동하기 시작합니다.

## routine 파일 살펴보기

자비스의 자동 routine은 `.claude/routines/` 폴더 안에 마크다운 파일 두 개로 정의돼 있어요.

| 파일 | 역할 |
|---|---|
| `daily-morning-brief.md` | 매일 평일 08:00 — `/morning-brief` 결과를 Gmail로 자기발송 |
| `weekly-review.md` | 매주 일요일 20:00 — `/weekly-review` 결과를 Gmail로 자기발송 |

각 파일 상단의 frontmatter에 `schedule:` 필드가 있는데, 이게 **cron 표현식**이에요. cron은 "매일 같은 시간에 자비스를 깨우는 알람"이라고 보시면 됩니다 — 1-2의 USB-C·OAuth 같은 표준 비유처럼, 시간 단위 자동 실행을 위한 업계 표준 표현 방식이에요.

```yaml
schedule: "0 8 * * 1-5"  # 평일(월~금) 08:00
schedule: "0 20 * * 0"   # 일요일 20:00
```

표현식 자체는 다섯 자리(분·시·일·월·요일)로 구성돼 있는데, 외우실 필요 없어요. 다음 단계에서 자비스에게 한국어로 말씀하시면 자동 변환됩니다.

## 시간 조정 (선택)

기본값(평일 08:00 · 일요일 20:00)이 본인 라이프스타일과 맞으면 그대로 두셔도 됩니다. 조정하고 싶으시면 다음 표를 참고하세요.

| 변경 의도 | cron 표현식 |
|---|---|
| 평일 07:30 | `30 7 * * 1-5` |
| 평일 09:00 | `0 9 * * 1-5` |
| 주말 포함 매일 08:00 | `0 8 * * *` |
| 일요일 회고 → 월요일 아침 07:00 | `0 7 * * 1` |

cron 표현식을 직접 적지 않으셔도 돼요. 자비스에게 한국어로 그대로 말씀하세요 — 예: "평일 8시 30분으로 바꿔주세요" — `.claude/routines/daily-morning-brief.md`의 schedule이 자동 업데이트됩니다.

기본값으로 가셔도 충분히 좋아요. 며칠 받아보시면서 본인 리듬에 안 맞으면 그때 다듬으시면 됩니다.

## routine 등록

이제 routine 파일을 클라우드에 올려 자동 작동을 시작합니다.

**[커맨드 입력]**

```
/3-4-register-daily-routine
```

자비스가 daily-morning-brief를 클라우드 routine으로 등록합니다. 등록이 끝나는 순간 자비스는 본인 노트북과 분리된 상태로 작동하기 시작해요.

**클라우드 routine이 가능한 이유** — Anthropic이 운영하는 클라우드 인프라가 cron 표현식대로 자비스를 깨우고, 페르소나 파일을 읽고, MCP를 통해 Gmail·Calendar에 접근해 메일을 발송합니다. 본인 노트북이 꺼져 있어도, 출장 중이어도, 휴가여도 매일 아침 8시에 메일은 어김없이 도착해요. 본인이 매일 클로드 코드를 켜야 할 필요가 없어집니다.

자비스가 다음과 같이 응답합니다.

```
✓ daily-morning-brief 등록 완료
  - 다음 실행: 2026-05-08 08:00 KST (내일)
  - cron: 0 8 * * 1-5
  - 발송 채널: Gmail 자기발송
```

같은 패턴으로 weekly-review도 등록하세요.

**[커맨드 입력]**

```
/3-4-register-weekly-routine
```

```
✓ weekly-review 등록 완료
  - 다음 실행: 2026-05-10 20:00 KST (이번 주 일요일)
  - cron: 0 20 * * 0
```

두 routine이 등록되면 자비스의 24/7 작동이 시작됩니다. 다만 다음 평일 8시까지 기다리지 않고, 지금 즉시 한 번 발송 테스트를 돌려볼 거예요.

## 즉시 발송 검증

이 단계가 워크숍의 마침표예요. 자비스가 routine을 즉시 1회 실행해서 본인 메일함에 사분면 정리 메일을 떨어뜨립니다.

**[커맨드 입력]**

```
/3-4-test-routine-now
```

자비스가 다음과 같이 진행 상황을 보여줍니다.

```
[자비스] daily-morning-brief 즉시 실행 중...
  ✓ 페르소나 로드
  ✓ Gmail 미답 메일 조회 (24h, N건)
  ✓ Calendar 오늘 일정 조회 (N건)
  ✓ 아이젠하워 매트릭스 분류 완료
  ✓ Gmail 발송 완료 → 요한님@gmail.com

다음 30초 안에 메일함을 확인하세요.
```

**그리고 본인 메일함을 여세요.**

받은편지함에서 다음을 확인하시면 됩니다.

- **보낸 사람** — 본인 (자기발송)
- **제목** — `[자비스] 오늘 — 아이젠하워 매트릭스 정리`
- **본문** — 프롤로그에서 보여드린 강사 자비스 메일과 같은 사분면 형식

메일을 열어서 세 가지를 가볍게 점검해주세요.

- **자비스 톤** — 페르소나 §3에서 정한 톤이 본문에 살아 있는가
- **사분면 형식** — Q1·Q2·Q3·Q4 + 시간 블록 추천이 페르소나 §5 보고 형식대로 정렬돼 있는가
- **본인 데이터 정확도** — Q1에 들어 있는 항목이 정말 오늘 우선이고, Q4가 정말 무시해도 되는 항목인가

세 가지가 OK이면 자비스 24/7 작동이 정말 시작된 거예요.

## 메일이 안 오면

처음엔 환경 차이로 막힐 수 있어요. 가장 흔한 문제는 **Gmail 스팸 분류**입니다 — 본인이 본인에게 보내는 메일이라 스팸으로 분류되는 경우가 있거든요.

::: details 흔한 문제와 해결
| 증상 | 해결 |
|---|---|
| 메일이 안 옴 | Gmail "스팸" 폴더 확인. 본인이 본인에게 보내는 메일이라 스팸 분류될 수 있음 — "스팸 아님" 표시 후 화이트리스트 등록 |
| "권한 부족" 에러 | Gmail 쓰기 권한 미추가 — 3-1로 돌아가 OAuth 동의에서 "메일 보내기" 체크 확인 |
| "페르소나 로드 실패" | `.claude/agents/자비스.md` 파일명 확인. 작명한 이름과 정확히 일치해야 함 |
| 빈 본문 | 페르소나 §4 Q1 규칙이 너무 좁음 — 자비스가 분류할 항목이 없음. 규칙을 한두 줄 더 풀어 다듬기 |
:::

여기서 막히시면 호스트(엘라)에게 바로 말씀해주세요. 워크숍 마지막 챕터가 코앞이라, 옆에서 도와드립니다.

---

::: tip 자비스 24/7 작동 시작
워크숍 4시간의 결승점이에요. **요한님의 자비스가 살아 있습니다.**

- 클라우드 routine 2종 등록 완료 (daily-morning-brief · weekly-review)
- 즉시 발송 테스트로 첫 메일 수신 확인
- 자비스 톤·사분면 형식·본인 데이터 정확도 검증

내일 아침 8시에 똑같은 메일이 또 도착합니다. 요한님은 아무것도 안 하셔도 돼요. 노트북을 켜지 않으셔도, 출장 중이어도, 휴가여도 — 자비스는 매일 같은 시간에 일어나 본인 데이터를 사분면으로 정리해 드립니다.

:::
