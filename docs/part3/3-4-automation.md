# 3-4. 자동 routine 등록 🎯 클라이맥스

::: info Step 12 / 13 · Part 3 · ⏱ 30분 · 🎯 클라이맥스
이 챕터를 마치면 — **자비스가 24/7 작동합니다. 매일 아침 8시·일요일 저녁 8시에 알아서 메일을 보내요. 그리고 이 챕터 끝에서, 30초 안에 첫 메일이 메일함에 떨어집니다.**
:::

## 0번 약속의 실현

[0. 오늘의 약속](/prologue)에서 강사 자비스의 메일을 보여드렸죠. **그게 30초 후 요한님 메일함에서 일어납니다.**

## 1. routine 파일 살펴보기 (5분)

`.claude/routines/` 폴더에 두 개의 routine 파일이 들어 있습니다:

| 파일 | 역할 |
|---|---|
| `daily-morning-brief.md` | 매일 평일 08:00 — `/morning-brief` 결과를 Gmail로 자기발송 |
| `weekly-review.md` | 매주 일요일 20:00 — `/weekly-review` 결과를 Gmail로 자기발송 |

각 파일 상단의 frontmatter에 `schedule:` 필드가 있어요. 이게 cron 표현식입니다.

```yaml
schedule: "0 8 * * 1-5"  # 평일 08:00
schedule: "0 20 * * 0"   # 일요일 20:00
```

## 2. 시간 조정 (선택, 5분)

기본값이 본인 라이프스타일과 맞지 않으면 조정하세요.

| 변경 의도 | cron 표현식 |
|---|---|
| 평일 07:30 | `30 7 * * 1-5` |
| 평일 09:00 | `0 9 * * 1-5` |
| 주말 포함 매일 08:00 | `0 8 * * *` |
| 일요일 → 월요일 아침 07:00 | `0 7 * * 1` |

자비스에게 대화로 요청하셔도 됩니다: "평일 8시 30분으로 바꿔주세요" → `.claude/routines/daily-morning-brief.md`의 schedule이 자동 업데이트.

## 3. routine 등록 (10분)

```
/3-4-register-daily-routine
```

자비스가 클라우드 routine을 등록합니다. 본인 노트북이 꺼져 있어도 매일 아침 8시에 클라우드에서 자비스가 깨어나 메일을 보냅니다.

**기대 결과**:

```
✓ daily-morning-brief 등록 완료
  - 다음 실행: 2026-05-08 08:00 KST (내일)
  - cron: 0 8 * * 1-5
  - 발송 채널: Gmail 자기발송
```

같은 패턴으로 weekly:

```
/3-4-register-weekly-routine
```

```
✓ weekly-review 등록 완료
  - 다음 실행: 2026-05-10 20:00 KST (이번 주 일요일)
  - cron: 0 20 * * 0
```

## 4. 즉시 발송 검증 ⭐⭐⭐ (10분)

::: tip 0번 약속의 실현
워크숍 4시간의 결산 순간입니다.

```
/3-4-test-routine-now
```

이 커맨드는 routine을 **즉시 1회 실행**해서 메일을 발송합니다. 다음 날까지 기다릴 필요가 없어요.
:::

**기대 결과 (콘솔)**:
```
[자비스] daily-morning-brief 즉시 실행 중...
  ✓ 페르소나 로드
  ✓ Gmail 미답 메일 조회 (24h, N건)
  ✓ Calendar 오늘 일정 조회 (N건)
  ✓ 아이젠하워 매트릭스 분류 완료
  ✓ Gmail 발송 완료 → 요한님@gmail.com

다음 30초 안에 메일함을 확인하세요.
```

**그리고 메일함을 여세요.**

📧 받은편지함에서 보낸 사람: 본인 / 제목: `[{{JARVIS_NAME}}] 오늘 — 아이젠하워 매트릭스 정리` 메일이 보여야 합니다.

::: tip ✅ 워크숍의 클라이맥스
이 메일이 0번 약속에서 보여드린 강사 자비스 메일과 같은 형태로 떨어졌나요?

그렇다면 — **요한님의 자비스가 살아 있습니다. 내일 아침 8시에 똑같은 메일이 또 도착합니다. 요한님은 아무것도 안 하셔도 돼요.**
:::

## 5. 메일이 안 오면

::: details 흔한 문제와 해결
| 증상 | 해결 |
|---|---|
| 메일이 안 옴 | Gmail "스팸" 확인. 본인이 본인에게 보내는 메일이라 스팸 분류될 수 있음 — "스팸 아님" 표시 후 화이트리스트 |
| "권한 부족" 에러 | Gmail 쓰기 권한 미추가 — 3-1 다시 |
| "페르소나 로드 실패" | `.claude/agents/{{JARVIS_NAME}}.md` 파일명 확인. 작명한 이름과 일치해야 함 |
| 빈 본문 | 페르소나 §4 Q1 규칙이 너무 좁음 — 자비스가 분류할 항목이 없음. 규칙 다듬기 |
:::
