# 3-3. 본인 옵션 채널 1개

::: tip Step 11 / 13 · Part 3
이 챕터에서 — **본인이 평소 쓰는 채널 1개를 자비스에 연결하고, 그 채널 전용 보조 커맨드를 함께 추가합니다.**
:::

## 왜 옵션 채널 1개인가

대표마다 핵심 채널이 달라요. Gmail · Calendar는 어느 분이든 공통이지만, 그 외에 매일 들여다보는 채널은 사람마다 다릅니다.

- **B2B 사업개발 중심** — LinkedIn DM·메시지가 영업·파트너십 채널의 핵심
- **팀 운영·실무 중심** — Slack · 카카오워크의 DM·멘션이 의사결정의 절반
- **문서·OKR 중심** — Notion 페이지에 OKR·할 일이 흩어져 있음

이 챕터에서는 본인의 **핵심 채널 1개만** 자비스에 연결합니다. 워크숍 4시간 안에 모든 채널을 다 연결하기보다, 가장 자주 쓰는 채널 하나를 자비스 운영 흐름에 단단히 묶는 쪽이 더 가치 있어요. 나머지 채널은 부록의 레시피로 본인이 필요할 때 자가 확장하실 수 있습니다.

## 본인 채널 선택

먼저 본인이 자비스에 어떤 채널을 연결하실지 자비스에게 알려주세요.

**[커맨드 입력]**

```
/3-3-connect-option-channel
```

자비스가 1-3 인터뷰의 "옵션 채널 1~2개" 답변을 바탕으로, 본인 직군·일하는 패턴에 맞는 채널을 추천합니다. 보통 다음 셋 중 하나로 안내됩니다.

| 채널 | 어울리는 분 | 난이도 | 패턴 |
|---|---|---|---|
| **Slack** | 팀 운영·실무 중심 | 하 | MCP 직결 (자동) |
| **Notion** | 문서·OKR·메모 중심 | 하 | MCP 직결 (자동) |
| **LinkedIn** | B2B 사업개발·네트워킹 중심 | 중 | 수동 + 초안 (반자동) |

추천 결과를 보시고 본인 분기를 결정하시면 됩니다.

## 본인 채널 연결

선택하신 채널에 따라 다음 분기 중 하나로 진행하세요. 본인 분기만 펼쳐서 따라가시면 됩니다.

::: details A. Slack — MCP 직결
Slack은 공식 MCP가 있어 자비스가 본인 워크스페이스에 직접 연결됩니다.

1. **Slack MCP 추가** — `/mcp` 화면에서 Slack 선택, 본인 워크스페이스로 OAuth 동의
2. **권한 범위 동의** — `channels:read` · `groups:read` · `im:read` · `chat:write` 네 가지 체크
3. **검증** — `/3-3-connect-option-channel`을 다시 실행해 연결 상태 확인

권한 범위 의미:
- `channels:read` · `groups:read` — 공개·비공개 채널 메시지 읽기
- `im:read` — 본인이 받은 DM 읽기
- `chat:write` — 답장 초안 작성 (자기 메시지 한정, Gmail 임시보관함과 같은 정책)

연결이 끝나면 자비스가 본인의 DM·멘션을 사분면으로 분류할 수 있어요.
:::

::: details B. Notion — MCP 직결
Notion은 공식 MCP가 있어 자비스가 본인 워크스페이스 페이지를 읽을 수 있습니다.

1. **Notion MCP 추가** — `/mcp` 화면에서 Notion 선택, 본인 워크스페이스로 OAuth 동의
2. **권한 범위 동의** — 읽기 + 페이지 생성 (메모만 자동 작성하시려면 읽기만 켜셔도 됩니다)
3. **검증** — `/3-3-connect-option-channel` 재실행

연결이 끝나면 자비스가 본인의 OKR·할 일·회의록 페이지를 읽어 `/morning-brief`에 통합할 수 있어요. Notion에 흩어진 할 일이 매일 아침 사분면 정리에 자연스럽게 합류합니다.
:::

::: details C. LinkedIn — 수동 + 초안
LinkedIn은 공식 MCP가 아직 없어 **수동 + 초안 패턴**으로 갑니다. 다른 두 채널처럼 완전 자동은 아니지만, 답장 초안 품질만큼은 페르소나 톤으로 일관되게 나와요.

1. **LinkedIn 메시지함을 텍스트로 추출** — 브라우저 확장 또는 수동 복사
2. **자비스에게 텍스트로 입력** — `/linkedin-triage` 호출 후 붙여넣기
3. **자비스가 메시지를 사분면으로 분류 + 답장 초안 작성** — 초안은 본인이 LinkedIn에 직접 붙여넣어 발송

자동화 정도는 낮지만, **답장 톤 통일**과 **사분면 정리**는 다른 채널과 동일한 품질로 받으실 수 있어요. 콜드 영업 메시지에 일관된 톤으로 빠르게 답할 수 있게 되는 효과가 가장 큽니다.
:::

## 옵션 채널 보조 커맨드

채널 연결이 끝났으면 그 채널 전용 운영 커맨드를 자비스에게 만들게 합시다. `/morning-brief` · `/inbox-triage`처럼 매일 호출할 도구가 한 가지 더 늘어나는 셈이에요.

**[커맨드 입력]**

```
/3-3-build-option-command
```

자비스가 본인이 선택한 채널에 맞는 보조 커맨드를 생성합니다. 채널별로 다음 형태입니다.

| 채널 | 커맨드 예시 | 자비스가 하는 일 |
|---|---|---|
| Slack | `/slack-triage` | DM·멘션을 한꺼번에 사분면으로 분류, Q1 항목은 답장 초안 작성 |
| Notion | `/notion-okr-check` | OKR·할 일 페이지를 읽어 `/morning-brief`에 통합, Q1 항목 자동 추출 |
| LinkedIn | `/linkedin-triage` | 메시지 텍스트 입력 → 사분면 분류 + 답장 초안 |

이 커맨드는 운영 커맨드 5종(`/morning-brief` · `/meeting-prep` · `/weekly-review` · `/inbox-triage` · `/meeting-recap`)과 함께 본인의 일상 운영 자산이 됩니다. 워크숍이 끝난 다음 날부터도 매일 호출하시는 도구예요.

## /morning-brief 통합 (선택)

옵션 채널 데이터를 매일 아침 브리핑에 함께 포함하시려면 페르소나 §1을 한 줄 보강하시면 됩니다.

```
연결된 옵션 채널: Slack (DM·멘션 우선), 매일 /morning-brief에 통합
```

페르소나에 이 한 줄을 더하면 `/morning-brief` 결과가 이렇게 바뀝니다.

- 오늘 사분면 정리 안에 **메일·일정 + 옵션 채널** 데이터가 함께 사분면으로 분류
- 예: Slack DM 중 Q1 항목이 메일 Q1과 같은 자리에 줄로 표시
- 시간 블록 추천도 옵션 채널 항목까지 고려해서 제안

통합하지 않으시면, 옵션 채널은 별도 보조 커맨드(`/slack-triage` 등)로 분리해서 따로 호출하실 수도 있어요. 본인 운영 스타일에 맞는 쪽으로 가시면 됩니다. 결정 어려우시면 일단 통합으로 가시고, 나중에 "분리해주세요" 한 줄로 페르소나를 다시 다듬으시면 됩니다.

페르소나 수정도 자비스에게 그대로 말씀하시면 자동 업데이트돼요.

---

::: tip 옵션 채널 연결 완료
본인 핵심 채널 1개가 자비스 운영 흐름에 들어왔어요.
- 옵션 채널 1개 OAuth 연결 (Slack · Notion · LinkedIn 중 하나)
- 채널 전용 보조 커맨드 1개 생성 (`/slack-triage` · `/notion-okr-check` · `/linkedin-triage`)
- (선택) `/morning-brief`에 옵션 채널 데이터 통합

운영 커맨드 5종 + 옵션 채널 보조 커맨드까지 — 자비스의 일상 운영 도구 세트가 완성됐습니다. 다음 챕터에서는 이 도구들을 자동 routine으로 등록해 24/7 작동시킵니다.
:::
