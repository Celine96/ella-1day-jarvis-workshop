# 3-3. 본인 옵션 채널 1개

::: info Step 11 / 13 · Part 3 · ⏱ 25분
이 챕터를 마치면 — **요한님이 평소 쓰는 채널 1개가 자비스에 연결되고, 그 채널 전용 보조 커맨드가 추가됩니다.**
:::

## 왜 옵션 채널인가

대표마다 핵심 채널이 달라요. Gmail·Calendar는 모두 공통이지만, 그 외엔:

- B2B 사업개발 중심 → **LinkedIn**
- 팀 운영·실무 중심 → **Slack** (또는 카카오워크)
- 문서·OKR 중심 → **Notion**

본인의 핵심 채널 **1개만** 이번 워크숍에서 연결합니다. 나머지는 부록의 레시피로 자가 확장.

## 1. 본인 채널 선택 (3분)
클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/3-3-connect-option-channel
```

자비스가 사전 체크리스트 답변을 보고 추천 채널을 안내합니다. 보통 다음 중 하나:

| 채널 | 추천 대표 | 난이도 |
|---|---|---|
| **Slack** | 팀 운영·실무 중심 | 하 (MCP 직결) |
| **Notion** | 문서·OKR·메모 중심 | 하 (MCP 직결) |
| **LinkedIn** | B2B 사업개발·네트워킹 중심 | 중 (수동 + 초안 위주) |

## 2-A. Slack 연결 (해당자만)

::: details Slack 연결 절차
1. Slack MCP 추가 → 본인 워크스페이스로 OAuth 동의
2. 권한 범위: `channels:read`, `groups:read`, `im:read`, `chat:write`
3. 검증: `/3-3-connect-option-channel` 재실행

자비스가 본인의 DM·멘션을 사분면으로 분류할 수 있습니다.
:::

## 2-B. Notion 연결 (해당자만)

::: details Notion 연결 절차
1. Notion MCP 추가 → 본인 워크스페이스로 OAuth 동의
2. 권한 범위: 읽기 + 페이지 생성 (읽기만 원하시면 읽기로 한정)
3. 검증

자비스가 본인의 OKR·할 일 페이지를 읽어서 morning-brief에 통합할 수 있습니다.
:::

## 2-C. LinkedIn 연결 (해당자만)

::: details LinkedIn 연결 절차
LinkedIn은 공식 MCP가 없어 **수동 + 초안** 패턴으로 갑니다.

1. LinkedIn 메시지함을 주기적으로 텍스트로 추출 (브라우저 확장 또는 수동 복사)
2. 자비스에게 텍스트로 입력
3. 자비스가 메시지를 사분면으로 분류 + 답장 초안 작성

자동화 정도는 낮지만, 답장 초안이 페르소나 톤으로 일관되게 나옵니다.
:::

## 3. 옵션 채널 보조 커맨드 (15분)
클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/3-3-build-option-command
```

자비스가 본인이 선택한 채널에 맞는 보조 커맨드를 생성합니다. 예시:

| 채널 | 커맨드 예시 | 역할 |
|---|---|---|
| Slack | `/slack-triage` | DM·멘션을 사분면으로 분류 |
| Notion | `/notion-okr-check` | OKR 페이지 읽어서 morning-brief에 통합 |
| LinkedIn | `/linkedin-triage` | 메시지 텍스트 입력 → 분류 + 초안 |

이 커맨드는 운영 커맨드 5종과 함께 평생 쓰는 자산이 됩니다.

## 4. /morning-brief 통합 (선택)

옵션 채널 데이터를 매일 아침 브리핑에 포함하시려면 페르소나 §1에 한 줄 추가:

```
연결된 옵션 채널: Slack (DM·멘션 우선), 매일 morning-brief에 통합
```

또는 자비스에게 대화로 말씀하시면 페르소나가 자동으로 업데이트됩니다.
