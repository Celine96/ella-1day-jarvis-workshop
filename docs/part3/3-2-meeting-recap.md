# 3-2. 회의록 + /meeting-recap

::: tip Step 10 / 13 · Part 3
이 챕터에서 — **본인이 평소 쓰는 회의록을 자비스에 연결해, 액션아이템 사분면 정리까지 함께 끝냅니다.**
:::

## 회의록 → 액션아이템 사분면 정리

회의록 자체를 만드는 일 — 음성을 녹음하고, 발화를 transcript로 옮기고, 요약을 정리하는 영역 — 은 이미 시장에 좋은 도구가 많아요. Tiro · Granola · Otter · Notta 같은 전사 도구가 그 일을 잘 합니다. 자비스의 가치는 회의록을 만드는 게 아니라 **그 다음 단계**예요.

| 단계 | 누가 |
|---|---|
| 회의 녹음·전사·요약 | 본인이 이미 쓰는 도구 (Tiro · Granola · Otter · Notta · 직접 작성 등) |
| **회의록 → 액션아이템 사분면 분류** | **자비스의 일** |

회의가 끝나면 보통 결정사항·액션아이템·미해결 질문이 머릿속에 흩어진 상태예요. 누구에게 무엇을 부탁할지, 본인이 7일 안에 할 일은 뭔지, 어느 항목은 폐기해도 되는지를 한 번 정리해야 회의가 진짜 마무리됩니다. 그 정리를 자비스가 사분면 형식으로 빠르게 끝내드립니다.

## 본인 회의록 도구 확인

먼저 본인이 어떤 도구로 회의록을 만드는지 자비스에게 알려주세요. 자비스가 도구별로 입력 방식을 추천합니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-connect-meeting-tool
```

자비스가 1-3 인터뷰 답변(§1 사용 도구)을 보고 다음 표대로 흐름을 안내합니다.

| 본인 도구 | 입력 방식 | 의미 |
|---|---|---|
| **Tiro** | A. MCP 직결 (자동) | 자비스가 Tiro에서 회의록을 직접 가져옴 |
| Granola · Otter · Notta · 기타 | B. 붙여넣기 | transcript 복사 후 자비스에게 전달 |
| 직접 작성 (메모장 · Notion 등) | B. 붙여넣기 | 회의록 텍스트 복사 후 전달 |
| 회의록 안 씀 | 이 챕터 스킵 | 회의록 도구 도입 후 다시 진행 |

본인 도구가 어디에 해당하는지 확인하시고 다음 단계로 가시면 됩니다.

## /meeting-recap 작동 설명서 확인

`/inbox-triage`와 같은 패턴이에요. 실행 전에 작동 설명서를 한 번 살펴보고 갑니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/3-2-build-meeting-recap
```

자비스가 `.claude/commands/meeting-recap.md`를 열어 핵심 동작 2가지를 보여줍니다.

1. **회의록 파싱** — 제목·일시·참석자·결정사항·액션아이템·미해결 질문을 자동 추출
2. **액션아이템 아이젠하워 분류** — 담당자·마감일을 기준으로 사분면 배치
   - Q1: 본인 담당 + 7일 이내
   - Q2: 본인 담당 + 8일 이상 또는 마감 미정
   - Q3: 타인 담당 — 위임 담당자 추천
   - Q4: 폐기 가능

작동 설명서 형식은 `/morning-brief` · `/inbox-triage`와 같은 구조예요. 마음에 안 드는 부분이 있으면 자비스에게 그대로 말씀하시면 자동 수정됩니다.

## 첫 실행

본인 도구에 따라 두 가지 흐름 중 하나로 실행합니다. 위 표에서 분기 결정하셨으면 본인 분기만 펼쳐서 따라가시면 돼요.

::: details A. Tiro 사용자 — MCP 직결
회의명을 인자로 넘기면 자비스가 Tiro에서 자동으로 회의록을 가져옵니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/meeting-recap "5/15 14:00 ABC테크 고객사 미팅"
```

자비스가 Tiro MCP를 통해 해당 회의록을 직접 조회·파싱합니다. 별도 붙여넣기 단계 없이 바로 결과가 나와요.
:::

::: details B. 다른 도구 사용자 — transcript 붙여넣기
회의록 도구(Granola · Otter · Notta · 메모장 등)에서 transcript를 복사한 뒤 다음을 입력하세요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/meeting-recap
```

자비스가 회의록 텍스트를 받을 준비를 합니다. 그대로 붙여넣으시면 자비스가 같은 흐름으로 파싱·분류합니다.
:::

자비스가 다음과 같이 출력합니다.

```
[자비스] 어제 ABC테크 고객사 미팅 회의록 정리

📅 일시: 2026-05-15 14:00~15:00
👥 참석자: 요한님, ABC테크 김CTO, ABC테크 이PM

## 결정사항
• 파일럿 도입 범위 확정 — 마케팅·세일즈 부서 우선
• 다음 마일스톤: 6월 첫 주 PoC 시작

## 액션아이템 (아이젠하워 분류)

🔴 Q1 — 요한님 본인이 7일 이내
• 파일럿 계약서 초안 작성 — 마감 5/22
• 김CTO에게 기술 통합 사전 자료 발송 — 마감 5/18

🟠 Q2 — 본인이 시간 확보 필요
• PoC 성공 지표 정의 — 마감 5/30

🟡 Q3 — 위임 (담당자 추천)
• 마케팅·세일즈 부서별 사용 시나리오 정리 → 담당 추천: 영업본부장
• 보안 실사 응답 자료 준비 → 담당 추천: 보안팀

⚪ Q4 — 스킵
• 명함 교환 정리 (자동)

## 미해결 질문
• 이PM이 짚은 SLA 응답 시간 정의 — 답변 마감 5/20
```

결과를 받으시면 두 군데를 차례로 확인해주세요.

- **Q1 본인 마감** — 본인이 7일 안에 처리해야 할 항목과 마감일
- **미해결 질문** — 회의에서 답을 못 정한 채 넘어간 질문, 답변 마감 챙기기

Q3 위임 항목은 자비스가 **담당자만 추천**해드립니다. 위임 메일·메시지 처리 방식은 두 가지 옵션이 있어요.

| 옵션 | 처리 방식 | 시점 |
|---|---|---|
| **A. 본인 직접 처리** | 자비스 추천 담당자를 보고 본인이 위임 메일·메시지 작성·발송 | 워크숍 안 흐름 (지금) |
| **B. 에이전트 자동화** (가능·제안) | 별도 에이전트가 자비스 추천을 받아 위임 메일 초안 작성·발송까지 자동화 | 워크숍 후 자가 개발 |

**B 옵션 — 에이전트 자동화**가 본인 업무 패턴에 맞는다면, 워크숍 후 부록 [§F. 에이전트 확장](/appendix#f-에이전트-확장) 가이드를 따라 자가 개발하실 수 있어요. 자비스의 인간 우선 원칙(외부 자동 발송 금지)을 보존하면서 위임 워크플로를 한 단계 더 자동화하는 패턴입니다.
