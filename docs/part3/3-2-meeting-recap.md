# 3-2. 회의록 + /meeting-recap

::: tip Step 10 / 13 · Part 3
이 챕터에서 — **본인이 평소 쓰는 회의록을 자비스에 연결해, 액션아이템 사분면 정리부터 Q3 위임 메일 초안 작성까지 함께 끝냅니다.**
:::

## 자비스는 회의록을 만들지 않습니다

회의록 자체를 만드는 일 — 음성을 녹음하고, 발화를 transcript로 옮기고, 요약을 정리하는 영역 — 은 이미 시장에 좋은 도구가 많아요. Tiro · Granola · Otter · Notta 같은 전사 도구가 그 일을 잘 합니다. 자비스의 가치는 회의록을 만드는 게 아니라 **그 다음 단계**예요.

| 단계 | 누가 |
|---|---|
| 회의 녹음·전사·요약 | 본인이 이미 쓰는 도구 (Tiro · Granola · Otter · Notta · 직접 작성 등) |
| **회의록 → 액션아이템 사분면 분류 + 위임 메일 초안** | **자비스의 일** |

회의가 끝나면 보통 결정사항·액션아이템·미해결 질문이 머릿속에 흩어진 상태예요. 누구에게 무엇을 부탁할지, 본인이 7일 안에 할 일은 뭔지, 어느 항목은 폐기해도 되는지를 한 번 정리해야 회의가 진짜 마무리됩니다. 그 정리를 자비스가 사분면 형식으로 빠르게 끝내고, 위임 항목은 후속 메일 초안까지 자동으로 작성해줍니다.

## 본인 회의록 도구 확인

먼저 본인이 어떤 도구로 회의록을 만드는지 자비스에게 알려주세요. 자비스가 도구별로 입력 방식을 추천합니다.

**[커맨드 입력]**

```
/3-2-connect-meeting-tool
```

자비스가 1-3 인터뷰 답변과 사전 체크리스트를 보고 다음 표대로 흐름을 안내합니다.

| 본인 도구 | 입력 방식 | 의미 |
|---|---|---|
| **Tiro** | A. MCP 직결 (자동) | 자비스가 Tiro에서 회의록을 직접 가져옴 |
| Granola · Otter · Notta · 기타 | B. 붙여넣기 | transcript 복사 후 자비스에게 전달 |
| 직접 작성 (메모장 · Notion 등) | B. 붙여넣기 | 회의록 텍스트 복사 후 전달 |
| 회의록 안 씀 | 이 챕터 스킵 | 회의록 도구 도입 후 다시 진행 |

본인 도구가 어디에 해당하는지 확인하시고 다음 단계로 가시면 됩니다.

## /meeting-recap 작동 설명서 확인

`/inbox-triage`와 같은 패턴이에요. 실행 전에 작동 설명서를 한 번 살펴보고 갑니다.

**[커맨드 입력]**

```
/3-2-build-meeting-recap
```

자비스가 `.claude/commands/meeting-recap.md`를 열어 핵심 동작 3가지를 보여줍니다.

1. **회의록 파싱** — 제목·일시·참석자·결정사항·액션아이템·미해결 질문을 자동 추출
2. **액션아이템 아이젠하워 분류** — 담당자·마감일을 기준으로 사분면 배치
   - Q1: 본인 담당 + 7일 이내
   - Q2: 본인 담당 + 8일 이상 또는 마감 미정
   - Q3: 타인 담당 → **후속 메일 초안 자동 작성**
   - Q4: 폐기 가능
3. **Q3 항목은 Gmail 임시보관함에 후속 메일 초안 저장** (외부로 나가지 않음)

작동 설명서 형식은 `/morning-brief` · `/inbox-triage`와 같은 구조예요. 마음에 안 드는 부분이 있으면 자비스에게 그대로 말씀하시면 자동 수정됩니다.

## 첫 실행

본인 도구에 따라 두 가지 흐름 중 하나로 실행합니다. 위 표에서 분기 결정하셨으면 본인 분기만 펼쳐서 따라가시면 돼요.

::: details A. Tiro 사용자 — MCP 직결
회의명을 인자로 넘기면 자비스가 Tiro에서 자동으로 회의록을 가져옵니다.

**[커맨드 입력]**

```
/meeting-recap "어제 투자자 미팅"
```

자비스가 Tiro MCP를 통해 해당 회의록을 직접 조회·파싱합니다. 별도 붙여넣기 단계 없이 바로 결과가 나와요.
:::

::: details B. 다른 도구 사용자 — transcript 붙여넣기
회의록 도구(Granola · Otter · Notta · 메모장 등)에서 transcript를 복사한 뒤 다음을 입력하세요.

**[커맨드 입력]**

```
/meeting-recap
```

자비스가 회의록 텍스트를 받을 준비를 합니다. 그대로 붙여넣으시면 자비스가 같은 흐름으로 파싱·분류합니다.
:::

자비스가 다음과 같이 출력합니다.

```
[자비스] 어제 투자자 미팅 회의록 정리

📅 일시: 2026-05-06 09:00~10:00
👥 참석자: 요한님, A 파트너, B 어소시에이트

## 결정사항
• Series A 라운드 사이즈 50억 → 70억으로 상향 검토
• 다음 마일스톤: ARR 50억 달성 (Q3 말)

## 액션아이템 (아이젠하워 분류)

🔴 Q1 — 요한님 본인이 7일 이내
• 업데이트된 IR 자료 작성 — 마감 5/12
• 핵심 고객 3명에게 레퍼런스 콜 부탁 — 마감 5/10

🟠 Q2 — 본인이 시간 확보 필요
• 70억 라운드 사이즈에 맞는 valuation 모델 재산정 — 마감 5/20

🟡 Q3 — 위임 (후속 메일 초안 작성됨)
• 데이터룸 업데이트 → 담당: 재무팀장, 임시보관함 ✓
• 법무 문서 1차 검토 → 담당: 외부 자문, 임시보관함 ✓

⚪ Q4 — 스킵
• A 파트너 명함 정리 (자동)

## 미해결 질문
• B 어소시에이트가 보낸 보안 실사 양식 — 답변 마감 5/15

후속 메일 초안 2건이 Gmail 임시보관함에 저장되었습니다. 발송 전 검토해주세요.
```

결과를 받으시면 세 군데를 차례로 확인해주세요.

- **Q1 본인 마감** — 본인이 7일 안에 처리해야 할 항목과 마감일
- **Q3 임시보관함 후속 메일 2건** — 위임 항목의 후속 메일 초안 (다음 섹션에서 검토)
- **미해결 질문** — 회의에서 답을 못 정한 채 넘어간 질문, 답변 마감 챙기기

## 후속 메일 검토

Gmail 좌측 메뉴에서 **"임시보관함(Drafts)"** 을 열어 자비스가 작성한 후속 메일 초안을 확인하세요. 3-1 답장 초안 검토와 같은 흐름입니다.

::: tip 후속 메일 패턴
- **수신자** — 위임받을 사람 (회의록 참석자 명단·페르소나 §2 팀 구성에서 자동 매칭)
- **제목** — `[{회의명} 후속] {액션아이템 요약}`
- **본문** — 회의 결정사항 인용 + 위임할 액션아이템 + 마감일 + 페르소나 §3 톤
:::

각 초안을 열어 한 번씩 훑어보세요. 마음에 들면 **"보내기"** 버튼만 누르시면 메일이 나갑니다. 마음에 안 들면 자비스에게 그대로 말씀하시면 됩니다 — 예: "재무팀장에게 보낼 데이터룸 메일은 마감일을 5/15로 바꿔주세요" — 자비스가 임시보관함의 그 초안을 다시 작성합니다.

자비스가 자동으로 외부 발송하는 일은 없어요. 모든 후속 메일은 본인이 검토하고 직접 "보내기"를 누를 때만 나갑니다 (3-1과 동일 정책).

::: tip /meeting-recap 첫 실행 완료
회의 한 건이 정리됐어요.
- 본인 회의록 도구 분기 확인 (MCP 직결 / 붙여넣기)
- `.claude/commands/meeting-recap.md` 작동 설명서 확인
- 액션아이템 사분면 분류 (Q1 · Q2 · Q3 · Q4)
- Q3 위임 항목의 후속 메일 초안 임시보관함 저장
- 외부 발송 0건

회의가 끝나도 흩어졌던 결정사항·액션아이템·위임 후속이 한 번에 정리되는 패턴을 손에 잡으셨어요. 다음 챕터에서는 본인이 평소 쓰는 옵션 채널 1개를 자비스에 추가합니다.
:::
