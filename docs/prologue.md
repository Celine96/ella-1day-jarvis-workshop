# 0. 오늘 만드는 자비스

::: tip 5분 · 워크숍 시작 직전
오늘 4시간 동안 **무엇을 만들고, 어떤 파일들이 손에 남는지** 먼저 보고 시작합니다.
:::

## 완성본 미리보기

워크숍이 끝나는 다음 날부터, **매일 아침 8시에 자비스가 본인 메일함으로 이런 메일을 보내줍니다.** 노트북을 켜지 않아도 자동으로요.

```
[자비스] 오늘 — 아이젠하워 매트릭스 정리

자비스가 오늘의 일정·할 일·메일 (총 14건)을
아이젠하워 매트릭스로 분류했습니다.

🔴 Q1 — Do (중요+긴급, 지금 처리)
• 09:00 투자자 미팅 — 어제 보낸 IR 자료 후속 답변 미수신 (8:30 안에 회신)
• 채용 후보자 A 최종 결정 — 이번 주 데드라인

🟠 Q2 — Schedule (중요+비긴급, 시간 블록 확보)
• 14:00 팀 위클리 — 어제 결정사항 3건 공유 준비 (13:30 30분)
• Q2 OKR 검토 — 금요일 전 (수요일 오후 추천)

🟡 Q3 — Delegate (비중요+긴급, 위임 추천)
• CFO 1Q 비용 승인 회신 → 재무팀장 1차 검토
• 슬랙 #product DM 3건 → 담당 PM 패스

⚪ Q4 — Delete (비중요+비긴급, 무시·삭제)
• 뉴스레터 3건 (읽기만)
• Linear 알림 12건 (휴지통)

추천 시간 블록
• 08:30~09:00 IR 후속 메일 답장 (10분)
• 13:30~14:00 위클리 자료 정리

⚠️ 일정 충돌: 11시 임원 콜과 12시 점심 30분 겹침
```

이 메일 한 통으로 **매일 아침 30분의 결정 비용**이 사라집니다.

## 실습 파일 — 자비스가 동작하는 구조

자비스는 다음 파일들로 만들어집니다. 워크숍 중 함께 만들어요.

| 폴더 / 파일 | 역할 | 만드는 시점 |
|---|---|---|
| `.claude/agents/{이름}.md` | **자비스 페르소나** — 이름·톤·우선순위 규칙·시간 블록 정책 | Part 1 (1-3, 1-4) |
| `.claude/commands/morning-brief.md` 외 4 | **운영 슬래시 커맨드 5종** — 일상 도구 | Part 2~3 |
| `.claude/routines/daily-morning-brief.md` 외 1 | **자동 실행 routine 2종** — 매일 아침·주간 자기발송 | Part 3-4 |
| `samples/inbox_sample.json` 외 1 | 학습용 더미 데이터 (본인 Gmail 연결 전 워밍업용) | 이미 준비됨 |

워크숍이 끝나면 위 파일들이 모두 손에 남고, 그대로 본인 일상에서 매일 작동합니다.

## 4시간 워크숍 구성

| 시점 | 단계 | 무엇을 하나 |
|---|---|---|
| 0:00 | **Prologue (5분)** | 지금 이 화면 |
| 0:05~1:00 | **Part 1 — 자비스 정체성 (55분)** | 작명 · 도구·우선순위 인터뷰 · 페르소나 자동 작성 |
| 1:00~2:30 | **Part 2 — 자비스 깨우기 (90분)** | 본인 Gmail/Calendar 연결 · `/morning-brief` 첫 실행 ⭐ |
| 2:30~4:00 | **Part 3 — 자비스 일상화 (110분)** | `/inbox-triage` ⭐ · 회의록 · 자동 routine 등록 🎯 |

⭐ = 본인 데이터로 첫 결과를 받는 순간 / 🎯 = 자비스가 24/7로 작동 시작

---

## 이렇게 진행돼요

각 챕터는 같은 패턴으로 구성됩니다 (ella-workshop과 동일).

- **[개념]** — 읽고 이해 (1~2분)
- **[실습]** — 슬래시 커맨드로 직접 만들기 + 본인 데이터 연결
- **[체크포인트]** — 진행 상황 확인 + 우와 모먼트
