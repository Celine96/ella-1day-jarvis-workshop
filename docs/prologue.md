# 0. 오늘의 약속

::: tip 5분 / 워크숍 시작 직전
이 챕터는 강사님과 함께 진행하실 거예요. 본인이 만들 자비스의 모습을 먼저 보고 시작합니다.
:::

## 1. 강사 자비스의 오늘 아침 메일 (1분)

강사님 메일함을 함께 봅시다. 오늘 아침 8시에 강사님의 자비스가 보낸 메일을 열어주세요.

::: details 미리보기 — 메일 본문 예시

```
[자비스] 오늘 — 아이젠하워 매트릭스 정리

자비스가 오늘의 일정·할 일·메일 (총 14건)을
아이젠하워 매트릭스로 분류했습니다.

🔴 Q1 — Do (중요+긴급, 지금 처리)
• 09:00 투자자 미팅 — 어제 보낸 IR 자료 후속 답변 미수신 (8:30 안에 회신)
• 채용 후보자 A 최종 결정 — 이번 주 데드라인

🟠 Q2 — Schedule (중요+비긴급, 시간 블록 확보)
• 14:00 팀 위클리 — 어제 결정사항 3건 공유 준비 (13:30 30분)
• Q2 OKR 검토 — 금요일 전 (수요일 오후 추천)
• 사외이사 디너 일정 후속 — 이번 주 안에 캘린더 픽스

🟡 Q3 — Delegate (비중요+긴급, 위임 추천)
• CFO 1Q 비용 승인 회신 → 재무팀장 1차 검토
• 슬랙 #product DM 3건 → 담당 PM 패스
• 외부 미디어 인터뷰 일정 조정 → 비서팀

⚪ Q4 — Delete (비중요+비긴급, 무시·삭제)
• 뉴스레터 3건 (읽기만)
• Linear 알림 12건 (휴지통)
• 콜드 영업 메일 2건 (휴지통)

추천 시간 블록
• 08:30~09:00 IR 후속 메일 답장 (10분)
• 13:30~14:00 위클리 자료 정리

⚠️ 일정 충돌: 11시 임원 콜과 12시 점심 30분 겹침
```

:::

## 2. 약속 (1분)

> **4시간 후, 요한님 메일함에서 똑같은 형태의 메일을 받게 됩니다.**

오늘이 끝날 때, 강사 자비스 메일과 같은 화면이 요한님의 메일함에 떨어집니다. 그게 이 워크숍의 도착점입니다.

## 3. 4시간 로드맵 (2분)

| 시점 | 단계 | 무엇이 일어나나 |
|---|---|---|
| **0:00** | 약속 | 지금 이 화면 |
| **0:30** | 미니 우와 | 자비스이(가) 요한님의 이름을 부르며 첫 인사 |
| **2:00** | 큰 우와 #1 | `/morning-brief` 첫 실행 — **본인 일정·할 일·메일을 아이젠하워 4사분면으로 분류** |
| **3:00** | 큰 우와 #2 | `/inbox-triage` — 미답 메일을 Q1 답변 / Q3 위임 / Q4 무시로 분류 |
| **4:00** | 🎯 클라이맥스 | **자동 routine 즉시 발송 검증** — 30초 안에 메일함 도착 |

## 4. 학습 모드 (1분)

::: warning 막혀도 괜찮습니다
- 프롬프트가 완벽하지 않아도 괜찮습니다. 자비스는 **클로드 코드와의 대화로 언제든 수정**할 수 있어요.
- 모르는 명령어가 나오면 그냥 멈추고 물어보세요.
- 본인 데이터로 직접 연결합니다. 민감 정보는 페르소나에 적지 마세요.
:::
