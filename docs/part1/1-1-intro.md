# 1-1. 자비스란 무엇인가

::: info Step 1 / 13 · Part 1 · ⏱ 10분
이 챕터를 마치면 — **자비스가 무엇이고, 무엇이 아닌지** 명확해집니다.
:::

## 시간관리의 진짜 페인

요한님의 하루를 떠올려 봅시다. 메일 50통, 일정 8건, 슬랙 알림 30개, 회의록 4건이 매일 쌓입니다.

진짜 문제는 **정보 부족**이 아닙니다. 정보는 이미 모든 도구에 다 있어요. 진짜 문제는:

> **"전부 봐야 하는데, 무엇에 먼저 시간을 쓸지 결정하는 비용."**

이 결정을 매일 아침마다 처음부터 다시 합니다. 30분이 사라집니다. 그게 누적되면 1년에 100시간이에요.

## 자비스의 정의

이 워크숍에서 만들 자비스는 단 한 가지를 잘하는 비서입니다.

> **시간관리 비서 — 일정·할 일·메일·회의록을 빠짐없이 모은 뒤, 아이젠하워 매트릭스로 분류·정렬해 의사결정 비용을 흡수하는 비서.**

콘텐츠 작성도, 이미지 생성도, 글쓰기도 하지 않습니다. **시간관리 한 가지만** 합니다.

## 분류 프레임워크 — 아이젠하워 매트릭스

자비스는 모든 항목을 4사분면으로 분류합니다.

| 사분면 | 정의 | 액션 라벨 |
|---|---|---|
| 🔴 **Q1** | 중요 + 긴급 | **Do** — 진행하기 |
| 🟠 **Q2** | 중요 + 비긴급 | **Plan** — 계획하기 |
| 🟡 **Q3** | 비중요 + 긴급 | **Delegate** — 위임 |
| ⚪ **Q4** | 비중요 + 비긴급 | **Delete** — 제거하기 |

이게 자비스의 정체성이자 워크숍 전체를 관통하는 언어입니다.

## TOP N이 아니라 "전체 + 정렬"

자주 오해하는 부분이 있어요. 자비스는 **"오늘 가장 중요한 3가지"**를 뽑아주지 않습니다.

| 잘못된 자비스 | 올바른 자비스 |
|---|---|
| ❌ "오늘 집중할 3가지: …" | ✅ "오늘 일정·할 일·메일 14건 전체. Q1 2건 / Q2 4건 / Q3 5건 / Q4 3건." |
| TOP N 추출 → 누락 우려 | 전체 빠짐없이 + 4사분면 태깅 |
| 요한님이 다른 항목을 별도로 점검 | 요한님은 사분면만 보면 됨 |

대표님 입장에서 누락은 가장 큰 리스크입니다. 자비스는 **빠짐없이 모은 뒤 정렬**할 뿐이에요. 무엇을 무시할지(Q4)도 명시합니다.

## 자비스가 하지 않는 것

::: warning 명시적 비기능
- ❌ 회의록을 직접 만들지 않음 — 외부 도구(Tiro·Granola·Otter 등) 결과를 입력으로 받습니다.
- ❌ 메일을 자동으로 발송하지 않음 — 답장 초안만 임시보관함에 저장합니다.
- ❌ 일정을 임의로 옮기지 않음 — 추천만 하고, 요한님이 직접 옮깁니다.
:::
