# 1-1. 자비스란 무엇인가

::: tip Step 1 / 13 · Part 1
이 챕터에서 — **자비스가 무엇이고 무엇이 아닌지** 우리가 함께 정의해봅시다.
:::

## 시간관리의 진짜 페인

요한님의 하루를 떠올려 봅시다. 메일 50통, 일정 8건, 슬랙 알림 30개, 회의록 4건이 매일 쌓입니다.

진짜 문제는 **정보 부족**이 아닙니다. 정보는 이미 모든 도구에 다 있어요. 진짜 문제는:

> **"전부 봐야 하는데, 무엇에 먼저 시간을 쓸지 결정하는 비용."**

이 결정을 매일 아침마다 처음부터 다시 합니다. 매일 30분, 매주 2~3시간이 결정 비용으로 사라집니다.

## 자비스의 기능 정의

오늘 만드는 자비스의 **5가지 핵심 기능**입니다.

1. **매일 아침 자동 브리핑**
   - 매일 아침 8시에 자비스가 오늘의 일정·미답 메일·할 일을 모두 모아 아이젠하워 매트릭스로 자동 분류한 뒤 본인 메일함으로 보내드립니다.
   - "언제 어떤 일을 하면 좋을지" 시간 블록을 추천하고, 일정이 겹치거나 하루가 너무 빡빡하면 경고로 알려드립니다.
2. **받은편지함 자동 정리**
   - 미답 메일을 모두 모아 사분면(지금 답할 Q1 / 위임할 Q3 / 무시해도 되는 Q4)으로 분류해드립니다.
   - Q1 메일에는 답장 초안까지 자비스가 미리 작성해 Gmail 임시보관함에 넣어둡니다. 검토 후 "보내기"만 누르시면 돼요.
3. **다음 회의 자동 준비**
   - 다가오는 회의의 참석자·지난 미팅 기록·관련 메일을 자동으로 모아 회의 맥락을 정리해드립니다.
   - 회의 안건도 사분면으로 분류하고, 본인이 미리 준비하시면 좋을 것들을 함께 알려드립니다.
4. **회의록 → 액션아이템**
   - 본인이 쓰시는 회의록 도구(Tiro·Granola·Otter 등)의 결과물을 자비스에게 그대로 넘겨주시면,
   - 액션아이템을 사분면으로 분류하고, 다른 분께 위임해야 할 항목(Q3)은 후속 메일 초안까지 자동으로 작성해드립니다.
5. **주간 회고·다음 주 우선순위**
   - 일요일 저녁 8시에 지난 한 주를 자비스가 회고해드립니다 — 완료한 일, 다음 주로 넘어가야 할 일, 시간 사용 패턴.
   - 다음 주 일정·우선순위를 미리 정리하고, 회의가 너무 몰린 날은 분산을 추천해드립니다.

### 전체 위계

```
자비스 (시간관리 비서)
├─ 핵심 기능 5개 ── 자비스가 매일/매주 직접 수행하는 액션
│   ├─ 매일 아침 자동 브리핑      (routine, 매일)
│   ├─ 받은편지함 자동 정리       (/inbox-triage, on-demand)
│   ├─ 다음 회의 자동 준비        (/meeting-prep, on-demand)
│   ├─ 회의록 → 액션아이템        (/meeting-recap, on-demand)
│   └─ 주간 회고·다음 주 우선순위 (routine, 매주)
│
├─ 확장 기능 1개 ── 학습자가 옵션으로 추가하는 통로
│   └─ 핵심 채널 연결 (LinkedIn/Slack/Notion 중 1개)
│
└─ 메타 2개 ── 자비스의 동작 원리·정책 (기능이라기보다 특성)
    ├─ 대화로 학습·진화 (자비스의 적응 능력 — 워크숍 전반에 깔린 원칙)
    └─ 본인 데이터 보호 (1-2 §5 보안 원칙 7가지에 이미 자세히 있음)
```

## 자비스의 인간 우선 원칙

- **회의록을 직접 만들지 않습니다** — 외부 도구(Tiro·Granola·Otter 등)의 결과를 입력으로 받습니다.
- **메일을 자동으로 발송하지 않습니다** — 답장 초안만 Gmail 임시보관함에 저장하고, 보내기는 요한님이 직접 누릅니다.
- **일정을 임의로 옮기지 않습니다** — 추천만 하고, 옮기는 결정은 요한님이 직접 합니다.

이 경계가 자비스의 신뢰감을 만듭니다.
