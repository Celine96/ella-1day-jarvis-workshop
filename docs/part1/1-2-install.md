## 클로드 코드란

자비스는 클로드 코드라는 작업실 안에서 살고 일합니다. 작업실에서 자비스는 요한님의 페르소나 파일을 읽고, 슬래시 커맨드(`/morning-brief` 등)를 실행하고, MCP를 통해 외부 도구(Gmail·Calendar)에 다녀옵니다.

## 설치 확인

::: tip 슬래시 커맨드로 검증
클로드 코드에서 다음을 입력하세요.

```
/1-2-install-check
```

**기대 결과**: 클로드 코드 버전과 사용자 환경 정보가 출력됩니다.
:::

이미 설치되어 있다면 30초면 끝나요. 처음이면 설치 가이드는 [클로드 코드 공식 안내](https://docs.claude.com/claude-code)를 참고해주세요.

## MCP — 약속·창구·출입증

MCP(Model Context Protocol)는 **자비스가 외부 도구에 접근할 수 있는 통로**입니다.

| 개념 | 비유 | v4에서 의미 |
|---|---|---|
| **약속** | "Gmail의 메일을 읽을 수 있게 한다" | MCP 서버 정의 — 어떤 작업이 가능한지 |
| **창구** | "Gmail 창구에 가서 신청서를 낸다" | MCP 호출 |
| **출입증** | "Gmail 계정 인증을 한 번 해둔다" | OAuth 인증 — 본인 계정 연결 |

자비스는 **본인의 출입증**으로 **본인의 창구**에 가서 **약속된 작업**만 합니다. 다른 사람 데이터에 접근하지 않고, 약속되지 않은 작업도 하지 않아요.

### v4에서 사용할 MCP

| MCP | 용도 |
|---|---|
| **Gmail** | 미답 메일 조회 · 답장 초안 · routine 자기발송 |
| **Google Calendar** | 일정 조회 |
| (옵션) Tiro | 회의록 가져오기 (3-2에서) |
| (옵션) Notion / Slack / LinkedIn | 학습자가 1개 선택 (3-3에서) |

본인 계정 인증은 **Session 2 시작 직후**에 진행합니다. 지금은 개념만 잡고 가면 됩니다.

## 자비스의 보안 원칙

본인 메일·일정에 자비스를 연결하는 일이라, 자비스를 설계하면서 보안 원칙을 시스템에 박아뒀습니다. 핵심 3가지:

### 발송은 절대 자비스가 하지 않습니다
- 답장 초안은 **Gmail 임시보관함**에 저장만 합니다
- "보내기" 버튼은 항상 요한님이 직접 클릭
- 자동 routine은 **본인 자기발송만** (외부 수신자 자동 발송 절대 금지)

### 권한은 언제든 1분 안에 회수
- Google 계정 보안 페이지에서 1분 안에 차단 가능
- 권한 회수 즉시 자비스는 본인 데이터에 접근 불가
- 자세한 회수 절차는 [부록 §B](/appendix#b-권한-회수-가이드)

### 결정은 항상 요한님
- 자비스의 사분면 분류·위임·삭제 추천은 모두 **추천**일 뿐
- 자비스는 결정 비용을 줄여드리지만, 결정 책임은 항상 본인
- 잘못된 추천을 따라가지 않을 자유가 보장됩니다

::: details 추가 보안 원칙 4가지 (펼쳐 보기)
- **단계별 권한 추가** — Session 2 읽기만 → Session 3 쓰기 추가 (자비스가 신뢰 가는 결과를 보여주기 전까진 쓰기 권한 X)
- **페르소나에 민감 정보 입력 금지** — 비밀번호·계좌·계약 금액·기밀 정보는 페르소나에 절대 입력 X
- **GitHub 공개 레포에 본인 데이터 안 올라감** — 워크북 콘텐츠·운영 코드만, OAuth 토큰은 클라우드 환경에서 격리 보관
- **분류가 틀리면 대화로 수정** — "이 발신자는 항상 Q4로요" → 페르소나 자동 진화

자세한 보안 체크리스트는 [부록 §D](/appendix#d-자비스-보안-체크리스트).
:::
