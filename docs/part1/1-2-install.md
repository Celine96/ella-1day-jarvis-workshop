## 클로드 코드란

자비스는 클로드 코드라는 작업실 안에서 살고 일합니다. 작업실에서 자비스는 요한님의 페르소나 파일을 읽고, 슬래시 커맨드(`/morning-brief` 등)를 실행하고, MCP를 통해 외부 도구(Gmail·Calendar)에 다녀옵니다.

## MCP — 자비스가 외부 도구에 접근하는 통로

### 왜 MCP가 필요한가 — LLM의 두 가지 한계

자비스의 '머리'에 해당하는 LLM(대규모 언어 모델)은 강력하지만 두 가지 한계가 있어요.

1. **학습 시점에 멈춰 있음** — LLM은 학습한 시점까지의 지식만 알 수 있어요. 어제 받은 메일이나 오늘 일정은 그 자체로는 알 수 없습니다.
2. **외부 도구를 직접 쓸 수 없음** — 메일을 보내거나, 캘린더에 일정을 등록하거나, Notion에 글을 쓰는 행동은 할 수 없어요.

이 두 한계를 풀려면 LLM에게 **외부 도구·실시간 데이터에 접근할 통로**가 필요합니다. 그게 **MCP(Model Context Protocol)** — Anthropic이 2024년 11월 만든 개방 표준이에요.

### MCP = AI 세계의 USB-C

비유하자면, 컴퓨터·휴대폰의 **USB-C 포트**와 같아요. USB-C 하나면 충전기·외장 디스크·모니터까지 모두 연결되듯이, MCP 하나로 자비스가 Gmail·Calendar·Notion·Slack 등 모든 도구에 **표준화된 방식**으로 연결됩니다.

이전엔 도구마다 별도의 연결 방식이 필요했지만, MCP 덕분에 **자비스 ↔ 모든 도구**가 한 통로로 깔끔하게 묶여요.

### 자비스가 사용할 MCP

이번 워크숍에서 자비스는 **Gmail · Google Calendar(필수)** 와 **요한님이 평소 쓰시는 옵션 채널 1개**를 MCP로 연결해 사용합니다. 자비스의 5가지 핵심 기능(매일 브리핑·받은편지함 정리·회의 준비·회의록 처리·주간 회고)이 모두 이 MCP들을 통해 실제 데이터를 보고 정리하게 되는 구조예요.

![자비스 + MCP 구조](/jarvis-mcp.png)

각 MCP의 역할과 사용 기능:

| MCP | 자비스 안에서 하는 일 | 사용 기능 |
|---|---|---|
| **Gmail** (필수) | 미답 메일 조회 · 답장 초안 임시보관함 저장 · 자동 routine 자기발송 | ①②③④⑤ (전체) |
| **Google Calendar** (필수) | 오늘·다음 주 일정 조회 · 회의 참석자·시간 분석 | ①③⑤ |
| (옵션) **Tiro · Granola · Otter** | 회의록 가져오기 (3-2에서 한 가지 연결) | ④ |
| (옵션) **Notion · Slack · LinkedIn** | 학습자가 평소 쓰는 채널 1개 (3-3에서 연결) | ① 통합 |

::: details 5가지 핵심 기능 펼쳐 보기
- ① 매일 아침 자동 브리핑
- ② 받은편지함 자동 정리
- ③ 다음 회의 자동 준비
- ④ 회의록 → 액션아이템
- ⑤ 주간 회고·다음 주 우선순위
:::

본인 계정 인증은 **Session 2 시작 직후**에 진행합니다. 지금은 개념만 잡고 가면 됩니다.

### MCP가 자비스에 가져다주는 3가지

- **실시간 정보** — LLM 학습 데이터가 아닌, 요한님의 **오늘 실제 메일·일정**을 직접 보고 분류합니다.
- **할루시네이션 감소** — 외부 데이터를 직접 가져오니, 자비스가 "있지도 않은 메일"을 만들어내지 않아요.
- **표준화된 안전성** — 본인 인증·권한 범위가 명확합니다. 자비스가 약속된 작업만 하고, 그 이상은 하지 않아요.

## 자비스의 보안 원칙

본인 메일·일정에 자비스를 연결하는 일이라, 보안 원칙을 둘로 나눠 짚어둡니다 — 자비스에 이미 박혀 있는 안전장치와 요한님이 직접 챙기시는 것.

### 시스템에 박혀 있는 안전장치

- **자비스는 메일을 직접 발송하지 않습니다.** 답장 초안을 작성하면 Gmail 임시보관함에만 저장해두니, 요한님이 검토하시고 "보내기" 버튼을 직접 누르셔야 메일이 나갑니다.
- **자비스의 권한은 단계적으로 늘어납니다.** 처음엔 메일·일정 읽기 권한만 받고, 자비스가 신뢰 가는 결과를 보여드린 뒤에야 답장 초안 저장이나 자기 발송 같은 쓰기 권한이 추가됩니다.
- **본인 데이터는 GitHub에 올라가지 않습니다.** 워크북 사이트 GitHub 레포에는 워크북 콘텐츠와 자비스 운영 코드만 올라가고, 요한님의 실제 메일·일정 데이터나 인증 토큰은 외부에 노출되지 않습니다.

### 요한님이 직접 챙기시는 것

- **모든 결정은 요한님이 직접 하시면 됩니다.** 자비스가 메일·일정을 사분면으로 분류하고 위임이나 삭제를 추천드리지만, 어디까지나 추천일 뿐입니다. 자비스의 추천이 마음에 안 들면 따라가지 않으셔도 됩니다.
- **페르소나 파일에 민감한 정보를 입력하지 마세요.** 비밀번호·계좌번호·계약 금액·기밀 정보 등은 페르소나에 절대 적지 마세요. 페르소나는 자비스의 분류 기준만 담는 곳입니다.
- **권한은 언제든 회수하실 수 있습니다.** 자비스 권한이 더 이상 필요 없으시면, Google 계정 보안 페이지에서 1분이면 차단할 수 있습니다 ([부록 §B](/appendix#b-권한-회수-가이드)). 회수하시기 전까지는 자비스가 평소대로 매일 아침 브리핑·routine을 보내드립니다.
- **자비스의 분류가 마음에 안 드시면 대화로 알려주세요.** "이 발신자는 항상 Q4로 분류해주세요" 같이 말씀하시면 페르소나가 자동으로 그 규칙을 학습해 다음부터 반영합니다.

이 두 축이 자비스를 안심하고 쓸 수 있게 만듭니다. 자세한 체크리스트는 [부록 §D](/appendix#d-자비스-보안-체크리스트).
