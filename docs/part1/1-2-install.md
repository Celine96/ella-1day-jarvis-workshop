# 1-2. 클로드 코드 + MCP

::: info Step 2 / 13 · Part 1 · ⏱ 25분
이 챕터를 마치면 — **클로드 코드가 설치되어 있고, MCP 개념과 자비스의 보안 약속 7가지가 손에 잡힙니다.**
:::

## 클로드 코드 = 자비스가 일하는 작업실

자비스는 클로드 코드라는 작업실 안에서 살고 일합니다. 작업실에서 자비스는:

- 요한님의 페르소나 파일을 읽고
- 슬래시 커맨드(`/morning-brief` 등)를 실행하고
- MCP를 통해 외부 도구(Gmail·Calendar)에 다녀옵니다.

## 1. 설치 확인

::: tip 슬래시 커맨드로 검증
클로드 코드에서 다음을 입력하세요.

```
/1-2-install-check
```

**기대 결과**: 클로드 코드 버전과 사용자 환경 정보가 출력됩니다.
:::

이미 설치되어 있다면 이 단계는 30초면 끝나요. 처음이면 설치 가이드는 [클로드 코드 공식 안내](https://docs.claude.com/claude-code)를 참고해주세요.

## 2. MCP란? — 약속·창구·출입증 비유

MCP(Model Context Protocol)는 **자비스가 외부 도구에 접근할 수 있는 통로**입니다.

비유로 풀면:

| 개념 | 비유 | v4에서 의미 |
|---|---|---|
| **약속** | "Gmail의 메일을 읽을 수 있게 한다" | MCP 서버 정의 — 어떤 작업이 가능한지 |
| **창구** | "Gmail 창구에 가서 신청서를 낸다" | MCP 호출 (`mcp__claude_ai_Gmail__search_threads` 등) |
| **출입증** | "Gmail 계정 인증을 한 번 해둔다" | OAuth 인증 — 본인 계정 연결 |

자비스는 **본인의 출입증**으로 **본인의 창구**에 가서 **약속된 작업**만 합니다. 다른 사람 데이터에 접근하지 않고, 약속되지 않은 작업도 하지 않아요.

## 3. v4에서 사용할 MCP

| MCP | 용도 | 권한 단계 |
|---|---|---|
| **Gmail** | 미답 메일 조회·답장 초안·routine 자기발송 | S2 읽기 → S3 쓰기 |
| **Google Calendar** | 일정 조회 | S2 읽기 |
| **(옵션) Tiro** | 회의록 가져오기 | S3-2에서 |
| **(옵션) Notion / Slack / LinkedIn** | 학습자 1개 선택 | S3-3에서 |

본인 계정 인증은 **Session 2 시작 직후**에 진행합니다. 지금은 개념만 잡고 가면 됩니다.

## 4. 권한 최소화 원칙

::: warning 권한은 단계별로 추가합니다
- **읽기부터** 시작 (Gmail·Calendar)
- 자비스가 신뢰 가는 결과를 보여주면 그때 **쓰기 추가**
- 워크숍 종료 후 권한 회수 가이드는 [부록 §B](/appendix#b-권한-회수-가이드)
:::

## 5. 자비스의 보안 약속 — 7가지

본인 메일·일정에 자비스를 연결하는 일이라, 자비스를 설계하면서 다음 7가지 원칙을 시스템에 박아뒀습니다. 워크숍 진행 전에 한 번 짚고 가시면 안심하고 시작하실 수 있어요.

### 5.1 발송은 절대 자비스가 하지 않습니다
- 자비스는 답장 초안을 **Gmail 임시보관함**에 저장만 합니다
- "보내기" 버튼은 항상 요한님이 직접 클릭
- 자동 routine은 **본인 자기발송만** (본인 메일함으로). 외부 수신자 자동 발송 **절대 금지**
- 이 원칙은 페르소나 §7에 명시적으로 박혀 있어 자비스가 자율적으로 풀 수 없습니다

### 5.2 권한은 언제든 1분 안에 회수
- 모든 권한(Gmail·Calendar·옵션 채널)은 **Google 계정 보안 페이지에서 1분 안에 차단** 가능
- 권한 회수 즉시 자비스는 본인 데이터에 더 이상 접근 불가
- 자세한 회수 절차는 [부록 §B](/appendix#b-권한-회수-가이드)

### 5.3 단계별 권한 추가 (신뢰 구축 후 확장)

| 시점 | Gmail | Calendar |
|---|---|---|
| Session 2 | 읽기만 | 읽기만 |
| Session 3 | + 쓰기 (임시보관함·자기발송) | 읽기 유지 |

자비스가 신뢰 가는 결과를 보여주기 전까지는 절대 쓰기 권한을 켜지 않습니다.

### 5.4 페르소나에 민감 정보 입력 금지
- **비밀번호·계좌·계약 금액·기밀 정보 등은 페르소나에 절대 입력 X**
- 자비스가 인터뷰 도중 그런 답변이 들어오면 자동으로 거절·재질문
- 페르소나에는 "분류 규칙·톤·우선순위 기준"만 담깁니다

### 5.5 GitHub 공개 레포에는 본인 데이터 절대 안 올라갑니다
- 워크북 사이트 GitHub 레포에는 **워크북 콘텐츠·자비스 운영 코드만** 올라갑니다
- 본인 Gmail·Calendar 실데이터, OAuth 토큰, 사용 기록은 **절대 GitHub에 올라가지 않습니다**
- routine은 클라우드 환경에서 인증 처리 — 코드와 분리되어 보관
- (필요시 레포를 처음부터 private로 만드시면 자비스 설정 자체도 비공개)

### 5.6 결정은 항상 요한님
- 자비스의 사분면 분류·위임 추천·삭제 추천은 모두 **추천**일 뿐
- "Q3 위임 → CFO에게"도 검토 후 본인이 결정
- 자비스는 결정 비용을 줄여드리지만, 결정 책임은 항상 본인. 잘못된 추천을 따라가지 않을 자유가 보장됩니다

### 5.7 분류가 틀리면 대화로 수정
- 자비스 분류가 마음에 안 들면 그대로 말씀하세요
- "이 발신자는 항상 Q4로요" → 페르소나 §4 자동 업데이트
- 한 달 쓰시면 본인에게 딱 맞게 진화 — 정확도가 시간이 갈수록 올라갑니다

::: tip 한 줄 요약
**자비스는 본인 데이터를 분류·정리해 추천만 합니다. 발송·결정은 항상 요한님이 직접.**
:::

::: warning 추가 점검 사항 (선택)
회사가 컴플라이언스 요구사항(SOC 2, ISO 27001, GDPR 등)이 있다면 워크숍 전에 다음을 점검하세요.
- 클로드(Anthropic) 데이터 처리 정책 — [Anthropic Privacy](https://www.anthropic.com/privacy)
- 회사 보안 정책상 외부 LLM에 메일 텍스트 전송 가능 여부
- 임시보관함 답장 초안에 들어갈 수 있는 정보 범위 (예: 고객 이메일 본문 인용)

회사 보안팀 검토가 필요하시면 워크숍 일정을 그 후로 잡으시는 것도 옵션입니다.
:::
