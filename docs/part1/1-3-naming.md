# 1-3. 자비스 만들기 : 작명과 인터뷰

::: tip Step 3 / 13 · Part 1
이 챕터에서 — **자비스에게 이름을 지어주고, 인터뷰로 컨텍스트를 받은 뒤 자비스의 첫 한 마디를 만나봅시다.**
:::

## 작명

자비스에게 이름을 지어주세요. **딱 1분**입니다. 길게 고민하지 마세요. 마음에 안 들면 나중에 바꿀 수 있어요.

작명 가이드:
- 짧게 (2~4자) — 매일 부를 이름이라 호흡이 길면 부담
- 부르기 좋게 — 발음이 자연스러운지
- 본인 컨텍스트 반영 OK — 회사명 변형, 본인 취향(영문/한글), 비서·집사 컨셉

::: tip 작명 도우미
회사 컨텍스트를 바탕으로 후보 3개를 제안받을 수 있어요. 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/1-3-name-helper
```

**기대 결과**: 회사 톤·역할·요한님의 취향에 맞는 자비스 이름 후보 3개.
:::

## 자비스 인터뷰

자비스가 요한님에게 직접 5가지를 묻습니다. 답변은 짧게, 본인 표현으로 하시면 됩니다. 사전에 무엇을 준비해 오실 필요 없어요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/1-3-inventory-import
```

자비스가 묻는 5가지:

| # | 질문 | 무엇을 답하나 |
|---|---|---|
| 1 | 회사 한 줄 소개 | 우선순위 판단의 가장 중요한 컨텍스트 (예: "B2B SaaS, ARR 30억, Series A 준비 중") |
| 2 | 사용 도구 | 메일·일정·회의록 도구 무엇을 쓰시는지 |
| 3 | 옵션 채널 1~2개 | 메일·캘린더 외 가장 자주 쓰는 채널 (LinkedIn / Slack / Notion 등) |
| 4 | **Q1 자동 분류 규칙 3개** | 무조건 최우선으로 잡아야 할 항목 — 예: "투자자·이사회 메일은 무조건 Q1" |
| 5 | **Q4 자동 분류 규칙 2~3개** | 안심하고 무시해도 되는 항목 — 예: "뉴스레터·콜드 영업" |

가장 중요한 건 4·5번입니다. **Q1 규칙은 요한님의 시간을 가장 비싸게 써야 하는 영역**, **Q4 규칙은 결정 비용을 가장 크게 줄이는 영역**이에요.

답변이 막연하면 자비스가 예시를 한 번 더 보여줍니다. 처음엔 부족해도 OK — 나중에 대화로 다듬을 수 있습니다.

답변이 끝나면 자비스가 정리한 표를 보여주고 확인받습니다. 다음 챕터(1-4)에서 이 답변이 페르소나 파일에 자동 반영돼요.

## 자비스의 첫 한 마디

::: tip 첫 만남
자비스가 살아 움직이기 시작하는 순간입니다. 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/1-3-first-hello
```

**기대 결과**:

> "요한님, 안녕하세요. 저는 **자비스** 입니다. 오늘부터 요한님의 시간관리를 도와드리겠습니다. 우선순위는 **아이젠하워 매트릭스**로 정리해드리고, 결정은 항상 요한님이 직접 하실 수 있도록 안내만 드립니다. 잘 부탁드립니다."
:::

이 한 줄이 자비스의 첫 발자국이에요. 마음에 안 드는 부분이 있다면 — 톤이 너무 격식 있다거나 부르는 호칭이 어색하다거나 — 그대로 자비스에게 말씀해보세요. **클로드 코드는 대화로 수정**합니다. 다음 챕터에서 페르소나를 본격 채울 때 다듬으면 됩니다.
