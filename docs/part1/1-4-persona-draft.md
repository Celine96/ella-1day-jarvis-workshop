# 1-4. 자비스의 매뉴얼 만들기

::: tip Step 4 / 11 · Part 1
이 챕터에서 — **1-3 인터뷰 답변에 추가 3가지 질문(톤·보고 형식·시간 블록)을 더해 자비스의 페르소나 파일을 함께 완성합니다.**
:::

## 왜 페르소나가 중요한가

페르소나 파일은 **자비스가 일을 시작할 때마다 펼쳐 보는 매뉴얼**이에요. `.claude/agents/자비스.md`라는 한 개의 마크다운 파일에 정체성·회사 컨텍스트·톤·우선순위 규칙·보고 형식을 모두 적어두면, 자비스가 매번 이 파일을 읽고 **같은 기준**으로 일해요. 어제와 오늘과 내일이 일관됩니다.

이렇게 하는 이유는 — AI는 본래 매 호출마다 정체성·기억이 새로 시작되기 때문이에요. 페르소나 파일이 없으면 자비스도 매번 처음 만나는 사람처럼 일을 시작해서, 어제 알려드린 회사 정보·말투를 오늘 또 알려드려야 합니다. 비서가 아니라 챗봇이 되는 거죠. 페르소나 파일에 한 번 적어두면 매번 같은 컨텍스트를 다시 전달할 필요가 없어집니다.

이번 챕터에서 자비스의 페르소나를 자동 작성합니다. 1-3 인터뷰 답변에 추가 3가지 질문(톤·보고 형식·시간 블록)을 더하면, 자비스가 직접 마크다운 파일에 기록해 줍니다.

특히 Part 3에서 등록할 **자동 routine** — 매일 아침 8시에 요한님 메일함으로 자동 발송되는 브리핑 — 도 이 페르소나를 읽고 작성됩니다. 노트북을 켜지 않으셔도, 페르소나에 적힌 톤·우선순위·보고 형식 그대로 자비스가 자동으로 메일을 보내드려요. **한 번 잘 채워두면 매일 일관된 비서가 됩니다.**

::: details 페르소나 파일은 어떻게 구성되나
8개 섹션(§1 ~ §7)으로 구성됩니다.
- **§1 정체성** — 자비스 이름, 역할 한 줄 정의
- **§2 회사 컨텍스트** — 1-3에서 받은 회사 한 줄 소개·사용 도구·옵션 채널
- **§3 톤** — 매일 보고 메일·인터랙션의 말투
- **§4 우선순위 분류 규칙** — 1-3에서 받은 Q1·Q4 자동 분류 규칙
- **§5 보고 형식** — 매일 메일을 어떤 형태로 받을지
- **§6 시간 블록 정책** — 회의·집중 시간대·버퍼 정책
- **§7 발송 채널 정책** — 어디로(메일·Slack 등) 발송할지

§1·§2·§4는 1-3에서 이미 받은 답변으로 자동 채워져요. 이 챕터에서는 §3·§5·§6를 추가로 채웁니다.
:::

## 페르소나 초안 자동 작성

학습자가 마크다운 파일을 직접 열어 편집할 필요는 **없습니다.** 자비스가 추가 3가지 질문을 차례로 물어보고, 답변을 받아 파일에 직접 기록합니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/1-4-persona-draft
```

자비스가 다음 단계로 진행합니다.

### 1단계 — 1-3 답변 자동 반영 (즉시)

1-3 인터뷰에서 받았던 답변이 페르소나 파일에 자동으로 들어갑니다. 별도 입력은 필요 없어요.

::: tip 매트릭스 다시 보기

| 사분면 | 의미 |
|---|---|
| 🔴 **Q1 — Do** | 중요 + 긴급, 진행하기 |
| 🟠 **Q2 — Plan** | 중요 + 비긴급, 계획하기 |
| 🟡 **Q3 — Delegate** | 비중요 + 긴급, 위임하기 |
| ⚪ **Q4 — Delete** | 비중요 + 비긴급, 제거하기 |
:::

| 페르소나 섹션 | 들어가는 내용 |
|---|---|
| **§2 회사 컨텍스트** | 회사 한 줄 소개 · 사용 도구 · 옵션 채널 후보 |
| **§4 Q1 (🔴 Do) 분류 규칙** | "무조건 최우선" 규칙 3개 (예: 투자자 메일·이사회 안건·중대 결정) |
| **§4 Q4 (⚪ Delete) 분류 규칙** | "안심하고 무시" 규칙 2~3개 (예: 뉴스레터·콜드 영업) |

이 단계에서 자비스가 짧게 정리해서 화면에 보여드려요. 빠진 게 없는지만 확인하시면 됩니다.

### 2단계 — 자비스의 추가 3가지 질문

자비스가 톤·보고 형식·시간 블록 — 이 세 가지를 차례로 물어봅니다. 매일 받게 될 브리핑 메일의 **말투·구조·시간 분배**를 지금 정해두는 단계예요.

**A. 톤**

> "제 톤은 어떻게 가져갈까요? A. 정중·효율 / B. 친근·간결 / C. 격식 — 또는 본인 톤을 자유롭게 한 줄로 알려주세요."

매일 아침 도착하는 자비스 메일의 말투가 여기서 결정됩니다. 같은 정보라도 톤이 다르면 받는 사람의 인상이 완전히 달라져요.

::: tip 톤 예시
- **A. 정중·효율** — "요한님, 다음과 같이 정리했습니다." → 비즈니스 컨텍스트, 빠르게 핵심으로
- **B. 친근·간결** — "오늘 하실 일이에요." → 일상적, 부담 없이
- **C. 격식** — "보고드립니다." → 공식 문서·이사회 톤
:::

지금 결정 못 하셔도 괜찮아요. 일단 하나 고르시고, 나중에 매일 메일을 받아보면서 "조금 더 친근하게" 같이 자비스에게 말씀하시면 톤이 자동으로 다듬어집니다.

**B. 보고 형식**

> "매일 받으실 메일은 어떤 형태가 편하세요?"

본인이 메일을 읽는 스타일에 맞춰 고르시면 됩니다. 헤드라인 먼저 보시는 분, 줄글로 쭉 읽으시는 분, 표로 한눈에 비교하시는 분 — 사람마다 다릅니다.

| 옵션 | 예시 | 어울리는 분 |
|---|---|---|
| **사분면 헤더 + 불릿** (추천) | 🔴 Q1 / • 항목 1 / • 항목 2 | 핵심을 먼저 스캔하고 싶은 분 |
| 산문 | "오늘은 ~한 일정이 있고, ~를 처리해야 합니다." | 줄글로 흐름을 따라가시는 분 |
| 표 | 사분면별 표로 정리 | 데이터 보듯 한눈에 비교하시는 분 |
| 혼합 | 사분면 헤더 + 불릿 + 시간 블록 표 | 정보가 많아도 다 보고 싶은 분 |

추천은 **사분면 헤더 + 불릿** — 프롤로그에서 보셨던 그 형태예요. 모바일에서도 가독성이 좋습니다.

**C. 시간 블록 정책**

자비스는 일정을 분류만 하는 게 아니라 **본인 업무 효율을 높이는 시간 블록**도 추천합니다. "이 시간에 이걸 하시는 게 좋아요" 같은 식으로요. 그 추천의 기본 틀이 시간 블록 정책이에요.

기본값은 일반적인 대표급 일정 패턴 — 오전은 의사결정·집중 업무, 오후는 외부 회의, 회의 사이엔 30분 버퍼(이동·정리 시간)를 두는 구조입니다. 본인의 실제 리듬과 다르시면 맞춰서 바꾸시면 됩니다.

예시:
- "저는 새벽이 집중 시간이라 06~09를 Q1·Q2로 잡아주세요"
- "점심 13~14는 절대 안 옮김 — 회의 잡지 마세요"
- "수요일 오후엔 외부 미팅 안 잡고, 사내 위클리만 받습니다"

이런 정책을 적어두면, 자비스가 매일 시간 블록을 추천할 때 본인 리듬을 어기지 않게 됩니다.

### 3단계 — 페르소나 파일 자동 저장

세 답변이 끝나면 자비스가 페르소나 파일을 **직접 작성·저장**합니다. 학습자가 마크다운 파일을 열거나 손으로 편집하실 필요가 전혀 없어요.

저장 위치: `.claude/agents/자비스.md` (자비스 작명에서 정한 이름이 파일명에 들어갑니다)

자비스가 작업하는 동안 화면에 진행 상황이 출력돼요 — "§3 톤 작성 중...", "§5 보고 형식 작성 중..." 같은 식으로요. 짧으면 30초, 길어도 1분 안에 끝납니다.

### 결과 확인

작성이 끝나면 자비스가 다음과 같이 출력합니다.

```
✅ 자비스의 페르소나가 완성됐습니다.

생성된 페르소나 파일: .claude/agents/자비스.md
- §1 정체성 ✓
- §2 회사 컨텍스트 ✓
- §3 톤 ✓
- §4 Q1·Q4 분류 규칙 ✓
- §5 보고 형식 ✓
- §6 시간 블록 정책 ✓
- §7 발송 채널 정책 (기본값) ✓
```

자비스가 만든 페르소나 본문도 화면에 함께 보여드려요. 한 번 쭉 훑으시면서 — 회사 컨텍스트가 잘 들어갔는지, 톤이 본인 표현 그대로인지, Q1/Q4 규칙이 빠진 게 없는지 — 가볍게 확인해주세요.

마음에 안 드는 부분이 있다면 자비스에게 그대로 말씀하시면 됩니다. **페르소나는 대화로 수정됩니다.**

예시:
- "Q1 규칙 #2를 '투자자 메일'로 바꿔주세요" → §4 자동 업데이트
- "톤을 조금 더 친근하게" → §3 자동 업데이트
- "보고 형식에 시간 블록 표도 추가해주세요" → §5 자동 업데이트

파일을 직접 편집하지 않아도, 다음 슬래시 커맨드를 실행할 때 자비스가 새 페르소나를 자동으로 반영합니다.

## Part 1 리캡

::: tip Part 1 완료
자비스가 태어났고, 페르소나가 완성됐습니다.
- 이름
- 회사 컨텍스트
- 톤
- Q1 / Q4 자동 분류 규칙
- 보고 형식
- 시간 블록 정책

페르소나 §8 자동화 정책은 비어 있어요. 자비스의 인간 우선 원칙 안전 기본값(수동) 상태입니다. 워크숍 후 본인 패턴에 맞춰 [부록 §F. 에이전트 확장](/appendix#f-에이전트-확장) 가이드로 자가 정의하실 수 있어요.

지금까지는 **"자비스가 어떤 비서일지"** 를 정의하는 단계였어요. 다음 Part에서는 자비스를 본인 Gmail · Calendar에 실제로 연결해 본인 데이터로 첫 일을 시키게 됩니다.
:::

Part 1에서 자비스의 **"정체성"** 을 잡았어요. 자비스가 무엇인지(1-1), 어떤 작업실에서 살고 어떤 안전장치 위에서 작동하는지(1-2), 본인 회사 컨텍스트와 우선순위 규칙은 어떻게 받았는지(1-3), 그 답변이 어떻게 페르소나 파일이 됐는지(1-4) — 자비스가 매번 같은 기준으로 일하기 위한 기반이 모두 이 Part에 만들어졌습니다.

| 챕터 | 한 줄 회고 |
|---|---|
| 1-1 자비스란 무엇인가 | 인간 우선 원칙 — 분류·작성은 자비스가, 결정·발송은 본인이 |
| 1-2 자비스의 작업실과 안전 약속 | MCP는 USB-C 같은 표준 통로 + 자비스의 보안 원칙 |
| 1-3 자비스 작명 + 첫 만남 | 자비스 이름 + 회사·우선순위 인터뷰 5가지 |
| 1-4 자비스의 매뉴얼 만들기 | 톤 · 보고 형식 · 시간 블록 정책 + 페르소나 파일 자동 작성 |

다음 Part에서는 자비스를 본인 Gmail · Calendar에 실제로 연결해 본인 데이터로 첫 결과를 받습니다. 자비스가 살아 움직이는 순간이에요.

쉬는 시간 짧게 갖고 — [Part 2 — 자비스 깨우기](/part2/2-1-persona-complete)로 가시죠.
