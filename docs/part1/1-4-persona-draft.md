# 1-4. 페르소나 자동 작성

::: info Step 4 / 13 · Part 1 · ⏱ 10분
이 챕터를 마치면 — **자비스가 1-3 답변 + 추가 3가지 질문(톤·보고 형식·시간 블록)으로 페르소나 파일을 자동 완성합니다. 학습자가 파일을 직접 열 필요 없습니다.**
:::

## 왜 페르소나가 중요한가

자비스가 매번 다른 톤으로 보고하면 비서가 아니라 챗봇이에요. 페르소나 파일은 자비스가 **항상 같은 톤·같은 우선순위 기준**으로 일하게 만드는 설계도입니다.

자동 routine이 매일 아침 8시에 발송되는 메일도 이 페르소나를 읽고 작성됩니다. 한 번 잘 채워두면 매일 일관된 비서가 됩니다.

## 페르소나 자동 작성 시작

```
/1-4-persona-draft
```

자비스가 다음을 자동 처리합니다.

### 1단계 — 1-3 답변 자동 반영 (즉시)

1-3 인터뷰에서 받은 답변이 페르소나 파일에 자동 들어갑니다:
- §2 회사 컨텍스트
- §4 Q1 자동 분류 규칙 (3개)
- §4 Q4 자동 분류 규칙

### 2단계 — 자비스의 추가 3가지 질문

자비스가 차례로 묻습니다.

#### A. 톤 (1분)
> "제 톤은 어떻게 가져갈까요? A. 정중·효율 / B. 친근·간결 / C. 격식 — 또는 본인 톤을 자유롭게 한 줄로."

::: tip 톤 예시
- **A. 정중·효율** — "요한님, 다음과 같이 정리했습니다."
- **B. 친근·간결** — "오늘 하실 일이에요."
- **C. 격식** — "보고드립니다."
:::

#### B. 보고 형식 (1분)
> "매일 받으실 메일은 어떤 형태가 편하세요?"

| 옵션 | 예시 |
|---|---|
| **사분면 헤더 + 불릿** (추천) | 🔴 Q1 / • 항목 1 / • 항목 2 |
| 산문 | "오늘은 ~한 일정이 있고, ~를 처리해야 합니다." |
| 표 | 사분면별 표 |
| 혼합 | 사분면 헤더 + 불릿 + 시간 블록 표 |

#### C. 시간 블록 정책 (2분)
> "기본값(오전 09~12 Q1·Q2 집중 / 오후 13~17 회의 / 회의 사이 30분 버퍼) 이대로 OK이신가요? 본인 라이프스타일에 맞게 바꾸셔도 됩니다."

예) "저는 새벽이 집중 시간이라 06~09를 Q1·Q2로", "점심 13~14는 절대 안 옮김".

### 3단계 — 페르소나 파일 자동 저장

자비스가 `.claude/agents/{{JARVIS_NAME}}.md` 파일을 직접 작성·저장합니다. 학습자가 파일을 열어 편집할 필요 없어요.

## 결과 확인

자비스가 다음을 출력합니다:

```

생성된 페르소나 파일: .claude/agents/{{JARVIS_NAME}}.md
- §1 정체성 ✓
- §2 회사 컨텍스트 ✓
- §3 톤 ✓
- §4 Q1·Q4 분류 규칙 ✓
- §5 보고 형식 ✓
- §6 시간 블록 정책 ✓

```

마음에 안 드는 부분이 있으면 자비스에게 그대로 말씀해보세요. **페르소나는 대화로 수정됩니다.** 예: "Q1 규칙 #2를 '투자자 메일'로 바꿔주세요" → 자동 업데이트.
