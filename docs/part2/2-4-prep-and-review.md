# 2-4. /meeting-prep + /weekly-review

::: info Step 8 / 13 · Part 2 · ⏱ 20분
이 챕터를 마치면 — **운영 커맨드 3종 (`/morning-brief` + `/meeting-prep` + `/weekly-review`)이 모두 작동합니다.**
:::

## 같은 패턴으로 두 개 더

`/morning-brief`를 만든 패턴과 똑같습니다. 자비스가 작동 설명서(`.claude/commands/*.md`)를 읽고, 본인 데이터로 결과를 출력해요.

## 1. /meeting-prep — 다음 회의 준비 (10분)
**[커맨드 입력]** 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/2-4-build-meeting-prep
```

자비스가 `.claude/commands/meeting-prep.md`를 함께 살펴봅니다. 그리고 바로 실행:
**[커맨드 입력]** 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/meeting-prep
```

**기대 결과**: 다음 회의 1건을 자동 선택해 다음을 정리합니다.

- 📅 시간·참석자·장소
- 컨텍스트 (지난 미팅·관련 메일 1~3줄)
- 안건의 아이젠하워 분류 (Q1 결정 / Q2 논의 / Q3 위임 / Q4 스킵)
- 요한님이 미리 준비할 것

::: tip 회의명 지정해서 호출
가장 가까운 회의 말고 특정 회의를 보고 싶으면:
**[커맨드 입력]** 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/meeting-prep "투자자 미팅"
```
:::

## 2. /weekly-review — 주간 회고 + 다음 주 (10분)
**[커맨드 입력]** 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/2-4-build-weekly-review
```

자비스가 `.claude/commands/weekly-review.md`를 함께 살펴봅니다. 그리고 실행:
**[커맨드 입력]** 클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

```
/weekly-review
```

**기대 결과**:

```
[자비스] 다음 주 — 아이젠하워 매트릭스 정리

## 지난 주 회고
- ✅ 완료한 Q1: N건
- ⚠️ 미완료 Q1 (carry-over): ...
- 📊 시간 사용: 회의 N시간 / 집중작업 N시간

## 다음 주 (월~일)
🔴 Q1 — Do
• [요일] [항목]
🟠 Q2 — Plan
• ...

## 추천 시간 블록 (주간)
- 월 09:00~11:00 [작업]
- ...

## 다음 주 부하 진단
- 화·수 회의 밀집 → 목요일 집중 시간 권장
- 일정 충돌: ⚠️ ...
```

이건 자동 routine으로 일요일 20:00에도 발송됩니다 (Part 3-4에서 등록).

## 3. 분류 미세조정

이 시점에서 자비스의 분류가 본인 일과 잘 안 맞는다고 느끼면, **페르소나 §4를 다듬는 게 가장 빠른 길**이에요.

흔한 사례:
- "Q3 위임 대상 추천이 부정확함" → 페르소나에 팀 구성·역할을 한 줄 추가
- "Q4가 너무 많음" → Q4 자동 분류 규칙을 더 구체적으로
- "주간 carry-over가 너무 공격적" → "carry-over는 마감 7일 이내만"으로 페르소나에 명시
