# 2-4. 회의 준비 + 주간 회고 — /meeting-prep · /weekly-review

::: tip Step 8 / 13 · Part 2
이 챕터에서 — **2-3에서 만든 패턴 그대로, 회의 준비와 주간 회고용 운영 커맨드 두 가지를 함께 추가합니다.**
:::

## 같은 패턴, 다른 시간 단위

2-3에서 `/morning-brief`를 만든 패턴 — 작동 설명서(`.claude/commands/{이름}.md`) 확인 → 본인 데이터로 실행 → 결과 보고 페르소나 다듬기 — 가 그대로 반복됩니다. 다만 **시간 단위가 달라요.**

| 커맨드 | 시간 단위 | 자비스가 보는 데이터 |
|---|---|---|
| `/morning-brief` (2-3) | 오늘 하루 | 오늘 일정·메일 |
| `/meeting-prep` | 다음 회의 1건 | 그 회의의 참석자·안건·관련 메일 |
| `/weekly-review` | 지난 주 + 다음 주 | 일주일치 일정·메일·회의 누적 데이터 |

세 가지가 갖춰지면 자비스의 **시간 시야가 하루·회의·일주일** 세 단위로 확장돼요. 매일 아침 브리핑은 오늘을, 회의 직전 준비는 한 번의 회의를, 일요일 회고는 주간 흐름을 — 모두 같은 페르소나 기준으로 정리해줍니다.

## /meeting-prep — 다음 회의 준비

회의 직전 5분에 자비스가 본인 대신 회의를 미리 훑어주는 커맨드예요. 학습자가 회의실 들어가기 전에 받아 한 번 읽으면, "이 회의에서 뭘 결정해야 하지?"가 한눈에 정리됩니다.

먼저 작동 설명서를 함께 살펴볼게요.

**[커맨드 입력]**

```
/2-4-build-meeting-prep
```

자비스가 `.claude/commands/meeting-prep.md`를 열어 어떤 흐름으로 회의를 준비하는지 보여줍니다. 핵심은 4가지 — 시간·참석자 기본 정보 / 회의 컨텍스트(지난 미팅·관련 메일) / 안건의 아이젠하워 분류 / 학습자가 미리 준비할 것.

확인이 끝나면 곧장 실행해보세요.

**[커맨드 입력]**

```
/meeting-prep
```

자비스가 다음 회의 1건을 자동으로 골라 다음을 정리합니다.

- **시간·참석자·장소** — Calendar에서 자동 추출
- **컨텍스트** — 지난 미팅 기록·관련 메일을 1~3줄로 요약
- **안건의 아이젠하워 분류** — Q1 결정해야 할 안건 / Q2 논의해야 할 안건 / Q3 위임 가능 / Q4 스킵해도 될 잡담성 안건
- **요한님이 미리 준비할 것** — 회의에서 본인이 답해야 할 질문, 미리 봐둘 자료 1~2개

결과를 받으시면 5분만 훑어보세요. 자비스의 정리가 본인 회의 감각과 맞으면 그대로 회의실 입장, 어긋나는 부분이 있으면 페르소나에 한두 줄 추가해주시면 다음 회의부터 더 정확해집니다.

::: tip 회의명 지정해서 호출
가장 가까운 회의 말고 특정 회의를 미리 보고 싶으시면, 회의 제목을 인자로 넘기시면 돼요.

**[커맨드 입력]**

```
/meeting-prep "투자자 미팅"
```
:::

## /weekly-review — 주간 회고와 다음 주 부하 진단

매주 일요일 저녁에 자비스가 한 주를 정리하고 다음 주를 예측하는 커맨드예요. 일요일 20시에 받아 30분 정도 읽고 다음 주 일정을 조정하시면, 월요일 아침을 훨씬 가볍게 시작할 수 있습니다.

먼저 작동 설명서부터.

**[커맨드 입력]**

```
/2-4-build-weekly-review
```

자비스가 `.claude/commands/weekly-review.md`를 열어 흐름을 보여줍니다. `/weekly-review`는 두 방향을 동시에 봐요 — **지난 주에 무엇이 끝났고 무엇이 넘어왔는지(회고)** 와 **다음 주 부하가 어떤지(예측)**.

곧장 실행해보세요.

**[커맨드 입력]**

```
/weekly-review
```

자비스가 다음과 같이 출력합니다.

```
[자비스] 다음 주 — 아이젠하워 매트릭스 정리

## 지난 주 회고
- ✅ 완료한 Q1: N건
- ⚠️ 미완료 Q1 (carry-over): ...
- 📊 시간 사용: 회의 N시간 / 집중작업 N시간

## 다음 주 (월~일)
🔴 Q1 — Do
• [요일] [항목]
🟠 Q2 — Plan
• ...

## 추천 시간 블록 (주간)
- 월 09:00~11:00 [작업]
- ...

## 다음 주 부하 진단
- 화·수 회의 밀집 → 목요일 집중 시간 권장
- 일정 충돌: ⚠️ ...
```

가장 유용한 부분은 **다음 주 부하 진단**이에요. 회의가 특정 요일에 몰려 있는지, 일정 충돌이 있는지, 어느 시간대에 집중 작업을 잡으면 좋은지를 자비스가 미리 짚어주거든요. 이 진단을 받고 일요일 저녁에 한 번만 일정 조정해두면, 한 주가 훨씬 가지런해집니다.

이 `/weekly-review`는 Part 3-4에서 자동 routine으로 등록됩니다. 그러면 매주 일요일 저녁 8시에 자동으로 본인 메일함에 발송돼요. 클로드 코드를 켤 필요 없이, 일요일 저녁에 메일 한 통이 도착하면 그게 다음 주의 시작점입니다.

## 분류 미세조정

운영 커맨드 3종을 모두 돌려보면, 2-3 첫 실행 때는 안 보이던 패턴이 새로 드러나요. 하루 단위(오늘 일정·메일)에서는 멀쩡했는데 회의 단위·주간 단위에서 어색해지는 분류가 있다면, 그건 페르소나 §4를 한 단계 더 정교하게 다듬을 단서입니다.

`/meeting-prep` / `/weekly-review`에서 흔히 발견되는 패턴 세 가지:

- **위임 대상 추천이 부정확함** — 자비스가 "이건 재무팀장에게" 같은 추천을 어색하게 한다면, 페르소나 §2(회사 컨텍스트)에 팀 구성·핵심 역할 한두 줄을 추가해주세요. 자비스가 위임 대상을 추론할 단서가 늘어납니다.
- **Q4가 너무 많음** — 자비스가 너무 많은 항목을 "무시해도 됨"으로 분류한다면, Q4 자동 분류 규칙을 더 구체적으로 풀어주세요. 예: "콜드 영업 메일 중 'introduce' · 'demo' 키워드 포함된 것만 Q4".
- **주간 carry-over가 너무 공격적** — 지난 주 미완료 항목을 다음 주로 너무 많이 넘긴다 싶으면, 페르소나 §4에 "carry-over는 마감일 7일 이내 항목만"처럼 임계값을 명시하시면 됩니다.

이렇게 한두 줄 보정한 뒤 다시 실행하시면, 두 번째 결과가 훨씬 본인 감각에 가깝게 나옵니다. 페르소나는 한 번에 완벽할 필요 없어요. **결과 → 다듬기 → 다시 실행 사이클을 두세 번 돌리는 게 자비스를 본인에게 길들이는 가장 빠른 방법**입니다.

## Part 2 리캡

Part 2에서 자비스가 본인 데이터에 처음 연결되어 **"읽고 정리하는 비서"** 로 작동하기 시작했어요. 페르소나를 마저 다듬고(2-1), 본인 Gmail · Calendar를 OAuth로 연결하고(2-2), `/morning-brief`로 본인 데이터에서 첫 결과를 받았고(2-3), 회의 준비·주간 회고용 운영 커맨드 두 개를 더 추가했습니다(2-4).

| 챕터 | 한 줄 회고 |
|---|---|
| 2-1 페르소나 점검·보강 | 회사 컨텍스트·시간 블록·발송 채널 정책 마감 |
| 2-2 MCP 연결 — 읽기 권한 | OAuth로 Gmail · Calendar 읽기 권한만 연결, 출입증 비유 |
| 2-3 자비스의 첫 결과 — /morning-brief | 본인 데이터에서 사분면 정리 첫 수신 |
| 2-4 회의 준비 + 주간 회고 — /meeting-prep · /weekly-review | 시간 시야가 하루 → 회의 → 일주일 세 단위로 확장 |

여기까지가 자비스의 **"읽고 정리하는"** 능력입니다. 본인 데이터를 보고, 사분면으로 분류하고, 시간 블록을 추천하는 부분이에요. 다음 Part에서는 자비스가 답장 초안 작성·routine 자동 발송까지 직접 손을 대는 **"실제로 일하는 비서"** 로 진화합니다.
