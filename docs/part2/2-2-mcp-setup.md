# 2-2. MCP 연결 — Gmail · Calendar 읽기 권한

::: tip Step 6 / 13 · Part 2
이 챕터에서 — **자비스를 본인 Google Calendar와 Gmail에 연결해, 첫 실시간 데이터를 함께 읽어봅니다.**
:::

## 권한은 "읽기"부터

자비스의 권한은 한 번에 다 주지 않고 **단계적으로 늘립니다.** 이 챕터에서는 Gmail·Calendar **읽기 권한만** 켭니다. 답장을 보내거나 일정을 수정하는 쓰기 권한은 Part 3의 첫 챕터(3-1)에서 자비스의 분류·작성 결과를 본인이 한 번 검증한 뒤에 추가해요.

| 시점 | Gmail | Calendar |
|---|---|---|
| **이 챕터 (2-2)** | 읽기 | 읽기 |
| Part 3 (3-1)부터 | 읽기 + 쓰기 | 읽기 유지 |

처음부터 쓰기를 주지 않는 이유는 1-2 보안 원칙에서 다뤘던 것과 같아요 — 자비스가 신뢰 가는 결과를 보여주기 전에 외부로 무엇이 나가지 않도록 만드는 안전장치입니다. 본인이 결과를 직접 보고 "이 정도면 답장도 맡길 수 있겠다"고 판단하신 뒤에 권한이 한 단계 올라가요.

## OAuth — 권한만 골라 위임하는 표준

본인 Google 계정을 자비스에 연결할 때 거치는 절차가 **OAuth**예요. 비밀번호를 직접 자비스에게 알려주는 게 아니라, **"자비스에게 메일 읽기만 허락한다"** 같이 권한 단위로 위임하는 표준 방식입니다.

1-2의 USB-C 비유에서 MCP가 자비스와 외부 도구를 잇는 통로라고 했는데, OAuth는 그 통로의 **출입증**이에요. 출입증에는 "어디까지 들어갈 수 있는지"가 적혀 있어서, 자비스는 그 범위 밖으로 한 발짝도 못 나갑니다. 또 출입증은 언제든 회수하실 수 있어요. 회수는 1분이면 끝나고, 절차는 [부록 §B](/appendix#b-권한-회수-가이드)에 있습니다.

## Google 계정 인증

클로드 코드 안에서 MCP 관리 화면을 여는 슬래시 커맨드부터 입력합니다.

**[커맨드 입력]**

```
/mcp
```

MCP 관리 화면이 열리면 다음 순서로 진행하세요.

1. **Google Calendar MCP 추가** — 목록에서 "Google Calendar" 선택, 본인 Google 계정으로 OAuth 동의 화면 진행
2. OAuth 동의 화면에서 **"캘린더 보기"** 만 체크 (수정·생성·삭제 권한은 끈 채로 진행)
3. **Gmail MCP 추가** — 같은 흐름으로 Gmail 선택, 본인 Google 계정 인증
4. OAuth 동의 화면에서 **"메일 읽기"** 만 체크 (보내기·수정·삭제 권한은 끈 채로 진행)

권한 범위는 OAuth 동의 화면 안에 체크박스 형태로 보여요. "메일 읽기"·"캘린더 보기" 외의 항목은 굳이 켜지 마세요. 다음 챕터에서 쓰기 권한이 필요해질 때 본인이 직접 다시 동의하시면 됩니다.

::: warning 본인 계정 직결 — 데이터 노출 주의
이 시점부터 자비스는 요한님의 진짜 메일·일정에 접근합니다.
- 페르소나 파일에 민감한 정보(비밀번호·계좌번호·계약 금액 등)는 절대 적지 마세요. 자비스가 페르소나를 매번 읽으니, 페르소나에 적힌 내용은 모두 LLM 처리 대상이 됩니다.
- 워크숍 종료 후 권한 회수는 [3-5 권한 회수](/part3/3-5-handoff)에서 안내합니다.
:::

## 연결 검증

권한 동의가 끝나면 자비스에게 연결 상태를 확인시켜봅시다. 본인 Google 계정의 첫 실시간 데이터를 받아보는 순간이에요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-2-mcp-test
```

자비스가 다음과 같이 출력합니다.

```
Gmail 연결 — 받은편지함 N통, 미답 N통 확인
Calendar 연결 — 오늘 일정 N건, 다음 7일 일정 N건 확인

샘플 출력:
- 가장 최근 미답 메일: [발신자] [제목]
- 다음 회의: [시간] [제목]
```

확인하실 부분은 두 가지예요.

- **카운트가 본인이 알고 있는 숫자와 비슷한가** — 메일·일정 개수가 너무 적거나 0이면 OAuth 권한 범위가 잘못 잡혔을 가능성이 큽니다. 인증을 다시 해주세요.
- **샘플 출력에 본인이 인식하는 메일·회의가 나오는가** — "가장 최근 미답 메일"과 "다음 회의" 한 줄을 보면, 자비스가 진짜 본인 데이터를 읽고 있는지 한눈에 확인됩니다.

이 두 가지가 맞으면 자비스가 요한님의 데이터를 읽을 준비가 끝났습니다. 다음 챕터(2-3)부터 자비스가 본인 데이터로 진짜 일을 시작해요.

## 인증이 안 되면

처음엔 OAuth 동의 화면이 막히거나 권한 범위가 어긋날 수 있어요. 흔한 에러와 해결책을 모아뒀습니다.

::: details 흔한 에러와 해결
| 에러 | 해결 |
|---|---|
| OAuth 동의 화면이 안 뜸 | 브라우저 팝업 차단 해제 후 `/mcp` 재시도 |
| 권한 범위 부족 | OAuth 동의 화면에서 "메일 읽기" / "캘린더 보기" 체크 누락 — 인증 다시 |
| 토큰 만료 | 다시 인증. Google은 일정 기간 미사용 시 자동 만료 |
| 한도 초과 (429) | 잠시 대기 후 재시도. 일반 사용에선 거의 발생 안 함 |
:::

여기서 막히시면 호스트(엘라)에게 바로 말씀해주세요. 워크숍 진행이 멈추지 않게 옆에서 도와드립니다.

## 데이터 안전 점검

연결 직후 자비스가 어디까지 닿을 수 있고, 어디는 절대 못 하는지 한 번 정리하고 가요. 1-2 보안 원칙에서 약속드린 경계를 실제 권한으로 확인하는 단계입니다.

자비스가 **지금 할 수 있는 것** — 메일 본문·발신자·제목·시간 읽기, 캘린더 일정·참석자·장소·설명 읽기.

자비스가 **지금 절대 못 하는 것** — 메일 발송·삭제·라벨 변경, 캘린더 일정 추가·수정·삭제.

읽기만 켜둔 상태라 자비스가 실수로든 의도로든 본인 메일·일정을 망가뜨릴 수 없습니다. 안심하고 다음 챕터로 넘어가셔도 돼요.
