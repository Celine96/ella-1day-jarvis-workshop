# 2-2. MCP 연결 — Gmail · Calendar (읽기)

::: info Step 6 / 13 · Part 2 · ⏱ 25분
이 챕터를 마치면 — **요한님의 Gmail과 Google Calendar에 자비스가 연결되어, 첫 데이터를 가져옵니다.**
:::

## 권한 단계 — 지금은 "읽기만"

| 시점 | Gmail | Calendar |
|---|---|---|
| **지금 (S2)** | 읽기 | 읽기 |
| 나중 (S3) | 쓰기 추가 | 읽기 유지 |

자비스가 신뢰 가는 결과를 보여주기 전까지는 **쓰기 권한을 주지 않습니다.** 이 챕터에서는 읽기만 켭니다.

## 1. Google 계정 인증 (10분)

::: tip 인증 절차
1. 클로드 코드에서 MCP 메뉴 열기
2. **Google Calendar** MCP 추가 → 본인 Google 계정으로 OAuth 동의 → 권한 범위는 **읽기만** 선택
3. **Gmail** MCP 추가 → 본인 Google 계정으로 OAuth 동의 → 권한 범위는 **읽기만** 선택
:::

권한 동의 화면에서 **"메일 읽기" / "캘린더 보기"**만 체크하시고, "보내기"·"수정"은 다음 단계까지 켜지 마세요.

::: warning 본인 계정 직결 — 데이터 노출 주의
이 시점부터 자비스는 요한님의 진짜 메일·일정에 접근합니다.
- 워크숍 종료 후 권한 회수 가이드는 [3-5 권한 회수](/part3/3-5-handoff)에서 안내합니다.
- 페르소나 파일에 민감 정보(비밀번호·계좌·계약 금액 등)는 절대 적지 마세요.
:::

## 2. 연결 검증 (10분)

```
/2-2-mcp-test
```

**기대 결과**: 자비스가 다음을 출력합니다.

```
✓ Gmail 연결 — 받은편지함 N통, 미답 N통 확인
✓ Calendar 연결 — 오늘 일정 N건, 다음 7일 일정 N건 확인

샘플 출력:
- 가장 최근 미답 메일: [발신자] [제목]
- 다음 회의: [시간] [제목]
```

이 출력이 나오면 자비스가 요한님의 데이터를 읽을 준비가 끝났습니다.

## 3. 인증이 안 되면

::: details 흔한 에러와 해결
| 에러 | 해결 |
|---|---|
| "OAuth 동의 화면이 안 뜸" | 브라우저 팝업 차단 해제 후 재시도 |
| "권한 범위 부족" | OAuth 동의 화면에서 "메일 읽기" / "캘린더 보기" 체크 누락 — 인증 다시 |
| "토큰 만료" | 다시 인증. Google은 일정 기간 미사용 시 자동 만료 |
| "한도 초과 (429)" | 잠시 대기 후 재시도. 일반 사용에선 거의 발생 안 함 |
:::

## 4. 데이터 안전 점검 (5분)

지금 자비스는 어디까지 볼 수 있는가:
- ✅ 메일 본문·발신자·제목·시간
- ✅ 캘린더 일정·참석자·장소·설명
- ❌ 메일 발송·삭제·라벨 변경 (쓰기 없음)
- ❌ 캘린더 일정 추가·수정·삭제 (쓰기 없음)

읽기만 켜둔 상태이므로, 자비스가 실수로 무엇을 망가뜨릴 수 없습니다.
