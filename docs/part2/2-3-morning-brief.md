# 2-3. /morning-brief 첫 실행

::: tip Step 7 / 13 · Part 2
이 챕터에서 — **자비스를 본인 일정·메일에 처음으로 풀어, 아이젠하워 매트릭스로 정리된 첫 결과를 함께 받아봅니다.**
:::

## 이 챕터의 의미

Part 1에서는 자비스의 정체성과 페르소나를 잡았고, 2-1·2-2에서는 본인 Google 계정에 연결해 자비스가 본인 데이터를 읽을 수 있게 만들었어요. 지금까지는 자비스에게 **"누구인지"** 와 **"출입증"** 을 줬다면, 이 챕터에서는 자비스가 본인 데이터로 **첫 일을 합니다.**

워크숍에서 처음으로 본인 데이터에서 의미 있는 결과를 받게 되는 단계예요. 결과가 본인 직관과 잘 맞으면 자비스를 일상화할 자신감이 붙고, 어긋나면 페르소나를 한 번 더 다듬으면 됩니다. 어느 쪽이든 다음 단계로 가는 데 필요한 정보가 손에 들어와요.

## 운영 커맨드 = 자비스의 작동 설명서

`/morning-brief`를 직접 실행하기 전에, 자비스가 그 커맨드를 받았을 때 **무엇을 어떤 순서로 하는지** 한 번 들여다볼게요. 자비스의 모든 슬래시 커맨드는 `.claude/commands/{이름}.md` 라는 마크다운 파일로 정의돼 있고, 자비스는 매번 이 파일을 읽고 그대로 일합니다. 페르소나 파일이 자비스의 **"성격"** 이라면, 운영 커맨드 파일은 자비스의 **"작동 설명서"** 예요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-3-build-morning-brief-step1
```

자비스가 `.claude/commands/morning-brief.md` 파일을 열어 5개 핵심 섹션을 함께 살펴봅니다.

| 섹션 | 자비스가 하는 일 |
|---|---|
| **§1 데이터 수집** | Calendar · Gmail · (옵션) 할 일 도구를 동시에 조회 |
| **§2 아이젠하워 분류** | 페르소나 §4의 Q1·Q4 규칙을 우선 적용해 사분면 배치 |
| **§3 시간 블록 추천** | 페르소나 §6의 시간 블록 정책에 맞춰 추천 슬롯 제안 |
| **§4 충돌·과부하 검증** | 일정 충돌 / Q1 과부하 / 회의 백투백을 점검해 알림 |
| **§5 출력 형식** | 페르소나 §5에 정한 형식(사분면 헤더 + 불릿)으로 정리 |

이 파일은 그대로 두셔도 되고, 마음에 안 드는 부분이 있으면 자비스에게 대화로 수정 요청하시면 돼요. 예: "Q1이 5건 넘으면 더 강하게 경고해주세요" — 자비스가 §4를 자동 업데이트합니다.

## 첫 실행

이제 자비스를 본인 데이터에 풀어볼 시간이에요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/morning-brief
```

자비스가 본인 일정·메일을 모두 읽고, 페르소나 규칙에 따라 사분면으로 분류해 다음과 같이 출력합니다.

```
[자비스] 오늘 — 아이젠하워 매트릭스 정리

자비스가 오늘의 일정·할 일·메일 (총 N건)을
아이젠하워 매트릭스로 분류했습니다.

🔴 Q1 — Do (중요+긴급, 진행하기)
• [본인 진짜 일정 1]
• [본인 진짜 메일 1]

🟠 Q2 — Plan (중요+비긴급, 계획하기)
• [본인 진짜 일정 2]

🟡 Q3 — Delegate (비중요+긴급, 위임 추천)
• [본인 진짜 메일] → [추천 위임 대상]

⚪ Q4 — Delete (비중요+비긴급, 제거하기)
• [본인 진짜 정보성 메일들]

추천 시간 블록
• HH:MM~HH:MM [작업명]

일정 충돌 (있으면): ...
```

이 결과가 자비스가 살아 움직이고 있다는 첫 증거예요. 본인의 진짜 일정·메일이 사분면으로 정렬되어 있을 거예요.

결과를 받으시면 두 가지를 천천히 확인해주세요.

- **분류가 본인 직관과 맞나요?** Q1에 들어 있는 항목이 정말 오늘 우선이고, Q4에 들어 있는 게 정말 무시해도 되는 것인지. 한두 개가 어긋나는 건 자연스러운 일 — 그 한두 개가 페르소나를 더 정교하게 다듬을 단서예요.
- **시간 블록 추천이 본인 일정과 호응하나요?** 추천 슬롯이 이미 잡혀 있는 회의 사이를 잘 피했는지, 본인 집중 시간대(페르소나 §6)와 일치하는지.

이 두 질문에 대부분 OK가 나오면 다음 단계로, 어긋나는 부분이 많으면 톤 튜닝과 분류 보정 단계에서 페르소나를 다듬어주세요.

## 톤 튜닝

자비스의 첫 결과를 읽으면서 "이 부분은 이렇게 바뀌면 좋겠다"는 게 보일 거예요. 자비스는 대화로 수정됩니다 — 파일을 직접 열어 편집하실 필요 없어요.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-3-build-morning-brief-step2
```

자비스가 어떤 부분을 어떻게 수정할지 묻고, 답하시는 대로 두 파일 중 한 곳을 자동으로 업데이트합니다.

| 수정 요청 종류 | 어디가 수정되나 |
|---|---|
| **분류 규칙 / 톤 / 회사 컨텍스트** | 페르소나 파일 (`.claude/agents/자비스.md`) |
| **출력 형식 / 임계값 / 데이터 수집 범위** | 운영 커맨드 (`.claude/commands/morning-brief.md`) |

::: tip 자주 나오는 튜닝 요청 예시
- "Q1이 너무 많아요. 5건 초과면 경고를 더 강하게 표시해주세요." → 운영 커맨드 §4 수정
- "Q4 항목은 발신자만 보여주고 제목은 생략해주세요." → 운영 커맨드 §5 수정
- "추천 시간 블록을 더 구체적으로요. 분 단위까지." → 운영 커맨드 §3 수정
- "투자자 메일은 항상 Q1 맨 위로요. 페르소나에 추가해주세요." → 페르소나 §4 수정
:::

수정이 끝나면 `/morning-brief`를 한 번 더 실행해서 결과가 어떻게 바뀌었는지 비교해보세요. 같은 데이터에서 톤·정렬·임계값이 달라진 게 보일 거예요.

## 분류가 이상하면

본인 회사·역할 컨텍스트와 자비스 분류가 어긋난다면, 그건 자비스가 잘못한 게 아니라 **페르소나가 충분히 구체적이지 않다는 신호**예요. 자비스는 페르소나 §4의 Q1·Q4 규칙을 1순위로 참조해 분류하는데, 규칙이 추상적일수록 자비스의 추론에 빈 자리가 생기거든요.

흔한 보정 예시:

- "이 발신자(예: 투자자 김OO)는 항상 Q1으로 잡아주세요" → 페르소나 §4 Q1 규칙에 이름 단위로 추가
- "Linear 알림 메일은 전부 Q4로 분류" → 페르소나 §4 Q4 규칙에 발신 도메인·키워드 추가
- "월요일 오전 위클리는 Q1이 아니라 Q2로" → 페르소나 §4 Q1 규칙에서 위클리 제외 명시

이렇게 한두 줄 보정한 뒤에 `/morning-brief`를 다시 실행하시면, 두 번째 결과는 첫 번째보다 훨씬 본인 직관에 가깝게 나옵니다. 자비스를 길들이는 가장 빠른 방법은 **결과를 보고 페르소나를 다듬는 짧은 사이클을 두세 번 반복**하는 거예요.
