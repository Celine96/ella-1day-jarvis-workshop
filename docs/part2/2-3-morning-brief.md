# 2-3. /morning-brief 첫 실행 ⭐ 큰 우와 #1

::: info Step 7 / 13 · Part 2 · ⏱ 30분 · ⭐ 큰 우와 #1
이 챕터를 마치면 — **요한님의 진짜 일정·할 일·메일이 아이젠하워 매트릭스로 정리된 첫 결과물이 손에 들어옵니다.**
:::

## 이 챕터의 의미

지금까지는 자비스에게 "정체성"과 "출입증"을 줬습니다. 이 챕터에서 자비스는 **본인 데이터로 첫 일을 합니다.** 워크숍의 첫 큰 성취예요.

## 1. 운영 커맨드 골격 확인 (10분)

```
/2-3-build-morning-brief-step1
```

**기대 결과**: `.claude/commands/morning-brief.md` 파일을 열어서 자비스가 함께 살펴봅니다. 이 파일은 자비스가 `/morning-brief`를 호출할 때마다 읽는 작동 설명서예요.

핵심 섹션:
- §1 **데이터 수집** — Calendar, Gmail, (옵션) 할 일 도구 병렬 조회
- §2 **아이젠하워 분류** — 페르소나의 Q1/Q4 규칙 우선 적용
- §3 **시간 블록 추천** — 페르소나 §6 정책
- §4 **충돌·과부하 검증** — ⚠️ 표시
- §5 **출력 형식** — 사분면 헤더 + 불릿

이 파일은 그대로 두셔도 되고, 자비스에게 대화로 수정 요청하셔도 됩니다.

## 2. 첫 실행 (15분) ⭐

::: tip 큰 우와 #1
이제 진짜 본인 데이터로 자비스를 깨워봅니다.

```
/morning-brief
```
:::

**기대 결과 (본인 메일·일정 기반)**:

```
[자비스] 오늘 — 아이젠하워 매트릭스 정리

자비스가 오늘의 일정·할 일·메일 (총 N건)을
아이젠하워 매트릭스로 분류했습니다.

🔴 Q1 — Do (중요+긴급, 진행하기)
• [본인 진짜 일정 1]
• [본인 진짜 메일 1]

🟠 Q2 — Plan (중요+비긴급, 계획하기)
• [본인 진짜 일정 2]

🟡 Q3 — Delegate (비중요+긴급, 위임 추천)
• [본인 진짜 메일] → [추천 위임 대상]

⚪ Q4 — Delete (비중요+비긴급, 제거하기)
• [본인 진짜 정보성 메일들]

추천 시간 블록
• HH:MM~HH:MM [작업명]

⚠️ 일정 충돌 (있으면): ...
```

**이 순간이 자비스가 살아 있다는 첫 증거입니다.** 본인의 진짜 일정·메일이 사분면으로 정렬되어 있어야 합니다.

## 3. 톤 튜닝 (5분)

```
/2-3-build-morning-brief-step2
```

결과를 보고 마음에 안 드는 부분이 있으면 자비스에게 그대로 말하세요.

::: tip 자주 나오는 튜닝 요청 예시
- "Q1이 너무 많아요. 5건 초과면 ⚠️ 경고를 더 강하게 표시해주세요."
- "Q4 항목은 발신자만 보여주고 제목은 생략해주세요. 너무 길어요."
- "추천 시간 블록을 더 구체적으로요. 분 단위까지."
- "투자자 메일은 항상 Q1 맨 위로요. 페르소나에 추가해주세요."
:::

자비스는 대화로 수정됩니다. 페르소나 파일이 자동으로 업데이트되거나, `.claude/commands/morning-brief.md`가 수정될 거예요.

## 4. 분류가 이상하면

본인 회사·역할 컨텍스트와 자비스 분류가 어긋나면 — 페르소나 §4 Q1·Q4 규칙을 더 구체적으로 다듬어주세요. 두 번째 실행에선 훨씬 나아집니다.
