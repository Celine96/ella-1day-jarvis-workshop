# 2-1. 페르소나 완성

::: tip Step 5 / 13 · Part 2
이 챕터에서 — **Part 1에서 자동 작성한 페르소나를 함께 점검하고, 빈 섹션이나 어색한 부분을 채워 자비스가 일관된 비서로 작동하도록 마무리합니다.**
:::

## Part 1에서 한 것 / Part 2에서 할 것

Part 1에서 자비스의 정체성과 페르소나 골격이 잡혔어요. Part 2에서는 그 페르소나를 본인 데이터(Gmail·Calendar)에 실제로 연결해서, 자비스가 살아 움직이게 만듭니다.

| 단계 | 한 것 / 할 것 |
|---|---|
| **Part 1 (완료)** | 자비스 작명 · 회사·우선순위 인터뷰 · 페르소나 자동 작성 |
| **Part 2 (지금부터)** | 페르소나 마무리 점검 · 본인 Gmail/Calendar 연결 · 운영 커맨드 3종 첫 실행 |

이 챕터(2-1)는 Part 2의 시작점 — **계정 연결 전에 페르소나가 정말 준비됐는지 확인하는 단계**예요. 페르소나가 빈약하면 자비스가 본인 데이터를 분류해도 결과가 어정쩡해지거든요. 5분만 들여 점검하고 가시면 다음 챕터부터의 결과가 훨씬 풍성해집니다.

## 페르소나 리뷰

1-4에서 자비스가 페르소나 파일을 자동으로 작성했지만, 인터뷰 답변이 짧았던 섹션이 있거나 1-4 시점엔 명확히 답하기 어려웠던 항목이 있을 수 있어요. 페르소나 리뷰는 자비스가 본인 페르소나 파일을 다시 훑으며 **"이 부분은 좀 더 구체적이면 좋겠어요"** 같은 식으로 짚어주는 단계입니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-1-persona-review
```

자비스가 페르소나 파일을 읽고, 비어 있거나 어색한 섹션을 짚어줍니다. 대표적으로 빠지기 쉬운 항목은 다음 4가지예요.

| 섹션 | 점검 포인트 |
|---|---|
| **§2 회사 컨텍스트** | 1~2줄로 명확한지 — "테크 회사"처럼 추상적이면 분류 정확도가 떨어집니다 |
| **§4 Q1 규칙** | 너무 추상적이지 않은지 — "중요한 일"보다 "투자자 메일"처럼 구체적으로 |
| **§4 Q4 규칙** | 자비스가 안심하고 무시할 만큼 구체적인지 — "잡다한 메일"보다 "뉴스레터·콜드 영업" |
| **§5 보고 형식** | 한 줄로 명시되어 있는지 — "사분면 헤더 + 불릿" 같이 |

자비스가 점검한 결과를 화면에 보여드리면, 짚어준 항목을 그대로 자비스에게 답변해주세요. 예: "Q1 규칙 #2를 '이사회 안건'으로 바꿔주세요." 자비스가 페르소나 파일을 자동으로 업데이트합니다.

## 회사 컨텍스트 보강

자비스가 메일·일정을 사분면으로 분류할 때 **회사 컨텍스트는 가장 먼저 참조하는 판단 근거**입니다. "이 메일이 정말 중요한가?"를 판단하려면 회사가 지금 어떤 단계에 있는지, 어떤 고객을 상대하는지를 알아야 하거든요.

1-3에서 한 줄로 받았던 회사 소개를 한 번 더 다듬을 시간이에요. 짧지만 구체적인 컨텍스트가 분류 정확도를 결정합니다.

::: tip 좋은 회사 컨텍스트 예시
- "B2B SaaS, ARR 30억 단계, Series A 준비 중. 고객사는 미드마켓~엔터프라이즈."
- "D2C 커머스, 월 매출 5억, 성장 정체기. 채널 다변화 검토 중."
- "AI 스타트업, Pre-A 라운드, 팀 12명. 엔터프라이즈 PoC 위주."

피해야 할 패턴:
- "테크 회사" — 너무 추상적, 자비스가 우선순위를 추론할 단서 부족
- "스타트업이에요" — 단계·규모·고객 정보가 없어서 판단 불가
:::

이 컨텍스트가 있어야 자비스가 "투자자 메일은 Q1, 채용 응대는 Q3" 같은 식의 일관된 분류를 매번 같은 기준으로 해드릴 수 있어요. 본인 회사를 한 줄로 어떻게 소개할지 떠오르지 않으면, 자비스에게 "회사 컨텍스트 후보 3개 제안해주세요"라고 말씀하시면 보강 후보를 만들어줍니다.

## 시간 블록 정책 점검

1-4에서 시간 블록 정책의 **기본 패턴**을 정의했어요. 2-1에서는 그 패턴이 본인 **실제 일주일 일정**과 맞는지 검증하는 단계입니다.

가볍게 머릿속으로 지난 한 주를 떠올리면서 다음 3가지를 확인해주세요.

- **오전(09~12) Q1·Q2 집중 시간이 본인 리듬과 맞나요?** 새벽이 집중 시간이라면 06~09로, 오후가 더 잘 맞으시면 14~17로 옮기시면 됩니다.
- **회의 사이 30분 버퍼가 적절한가요?** 빡빡한 일정을 선호하시면 15분으로 줄이거나, 회복 시간이 더 필요하시면 45분으로 늘리실 수 있습니다.
- **고정 일정이 있으신가요?** 점심·운동·정기 회의 등 절대 옮기지 않는 슬롯이 있다면 명시하시면 좋아요. 예: "13:00~14:00 점심 절대 안 옮김", "수요일 오전 임원 위클리 고정".

수정사항을 자비스에게 그대로 말씀하시면 §6이 자동 업데이트됩니다. 예: "오전 집중 시간을 06~09로 바꿔주세요" — 페르소나 자동 반영.

## 발송 채널 정책

페르소나 §7에는 발송 채널 정책이 들어갑니다. 자비스가 어떤 메일은 자동으로 보낼 수 있고, 어떤 메일은 절대 자동으로 보내면 안 되는지의 경계예요. 이 경계가 페르소나에 박혀 있어야 자비스가 매번 같은 안전 기준으로 작동합니다.

기본 정책은 다음과 같이 두 갈래로 나뉘어요.

| 구분 | 발송 허용 | 동작 방식 |
|---|---|---|
| **자기발송** (요한님 → 요한님) | 허용 | routine 산출물(예: 매일 아침 브리핑)을 본인 메일함으로 보내는 것 |
| **외부 발송** (요한님 → 외부) | 금지 | 답장·신규 메일 등 외부로 나가는 것은 모두 Gmail 임시보관함에만 저장 |

::: warning 왜 이 경계가 필요한가
자비스가 분류한 결과는 매우 정확하지만, **100%는 아닙니다.** 만약 자비스가 잘못 분류한 답장이 외부로 자동 발송된다면 — 투자자에게 잘못된 답이 가거나 고객에게 톤이 어긋난 메시지가 가면 — 되돌릴 수 없어요.

임시보관함에 저장하는 방식은 그 위험을 0으로 만듭니다. 자비스가 답장 초안을 만들고 "검토 요청" 메모와 함께 임시보관함에 넣어두면, 본인이 한 번 훑고 "보내기" 버튼을 직접 누를 때만 메일이 나갑니다. 자비스의 분류·작성은 빠른 속도로, 발송 결정은 본인이 — 이 분리가 자비스를 안심하고 쓸 수 있게 만드는 핵심이에요.

자기발송은 본인 메일함 안에서만 움직이니 외부에 전혀 영향 없고, 그래서 routine 자동화에 한해 허용합니다.
:::

페르소나 §7에 이 정책이 박혀 있는지만 확인하시면 됩니다. 자비스가 점검 단계에서 자동으로 검증해드려요.

## 페르소나 저장 + 다시 인사

점검·보강이 끝나면 페르소나 파일을 저장하고, 자비스에게 다시 인사를 시켜봅시다. 1-3에서 받았던 첫 인사는 페르소나 §1·§2·§4의 일부만 반영된 상태였어요. 지금은 §1~§7이 모두 채워진 **풀 페르소나** 상태라, 인사 한 마디에서도 자비스가 본인 컨텍스트를 훨씬 풍성하게 담아냅니다.

클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-1-persona-greeting
```

기대 결과 예시:

> "요한님, 페르소나 정리가 끝났습니다. 저는 **자비스**입니다. B2B SaaS · Series A 준비 단계 컨텍스트에 맞춰, 투자자·이사회 메일을 Q1으로 자동 분류하고, 뉴스레터·콜드 영업은 Q4로 정리해 드리겠습니다. 답장 초안은 항상 임시보관함에 저장해 요한님이 검토하신 뒤 직접 보내실 수 있게 안내드립니다. 잘 부탁드립니다."

회사 컨텍스트·우선순위 규칙·발송 정책이 한 인사에 모두 녹아 있는 게 보이실 거예요. 1-3 때의 짧은 인사와 비교하시면 페르소나가 자비스를 어떻게 바꿨는지 한눈에 느껴집니다.

마음에 안 드는 부분이 있으면 그대로 말씀하시면 돼요 — "톤을 조금 더 친근하게", "이사회 안건도 Q1에 명시해주세요" — 페르소나가 자동으로 다듬어집니다.

---

::: tip 페르소나 완성
페르소나 파일이 모든 섹션 다 채워졌습니다.
- §1 정체성 (자비스 이름)
- §2 회사 컨텍스트 (구체 1~2줄)
- §3 톤
- §4 Q1·Q4 분류 규칙
- §5 보고 형식
- §6 시간 블록 정책 (본인 실제 일정 반영)
- §7 발송 채널 정책 (자기발송 허용 / 외부 발송 차단)

자비스가 매일 일관된 톤·우선순위로 일할 준비가 끝났어요. 다음 챕터에서는 Gmail·Calendar 본인 계정을 자비스에게 연결합니다. 자비스가 본인 데이터를 처음 만나는 순간이에요.
:::
