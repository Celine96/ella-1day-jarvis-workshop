## 1. 페르소나 리뷰 (5분)
클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/2-1-persona-review
```

**기대 결과**: 페르소나 파일에서 비어있거나 어색한 섹션을 자비스가 짚어줍니다.

대표적으로 빠지기 쉬운 항목:
- **§2 회사 컨텍스트** — 1~2줄로 명확한지
- **§4 Q1 규칙** — 너무 추상적이지 않은지 (예: "중요한 일" → ❌, "투자자 메일" → ✅)
- **§4 Q4 규칙** — 자비스가 안심하고 무시할 만큼 구체적인지
- **§5 보고 형식** — 한 줄로 명시되어 있는지

## 2. 회사 컨텍스트 보강 (3분)

자비스가 일정·메일을 분류할 때 **회사 컨텍스트가 가장 중요한 판단 근거**입니다.

::: tip 좋은 회사 컨텍스트 예시
- "B2B SaaS, ARR 30억 단계, Series A 준비 중. 고객사는 미드마켓~엔터프라이즈."
- "D2C 커머스, 월 매출 5억, 성장 정체기. 채널 다변화 검토 중."
- "AI 스타트업, Pre-A 라운드, 팀 12명. 엔터프라이즈 PoC 위주."

❌ 너무 추상적: "테크 회사"
:::

자비스가 "이 메일이 정말 중요한가?"를 판단할 때 회사 컨텍스트를 1순위로 참조합니다.

## 3. 시간 블록 정책 점검 (3분)

페르소나 §6은 기본값이 채워져 있어요. 본인 라이프스타일과 맞는지 확인:

- 오전(09~12): Q1·Q2 집중 → 본인은 오전이 집중 시간인가? 아니면 오후?
- 회의 사이 30분 버퍼 → 빡빡한 일정을 선호하면 15분으로 줄이거나
- 점심·운동 등 고정 일정이 있으면 명시 (예: "13:00~14:00 점심 절대 안 옮김")

## 4. 발송 채널 정책 (2분)

페르소나 §7 "메일 자동 발송 절대 금지"가 박혀 있는지 확인합니다. 자비스가 routine으로 자기발송하는 건 허용하지만, **답장이나 외부 발송은 항상 임시보관함에만 저장**합니다.

::: warning 발송 권한의 경계
- ✅ 자비스 → 요한님 (자기발송, routine 산출물)
- ❌ 자비스 → 외부 (답장·메시지 자동 발송)

이 경계는 워크숍 내내 절대 풀리지 않습니다.
:::

## 5. 저장 + 다시 인사

페르소나 파일을 저장하고 자비스에게 다시 인사를 시켜보세요.
클로드 코드 터미널에서 다음 슬래시 커맨드를 입력하세요.

**[커맨드 입력]**

```
/1-3-first-hello
```

**기대 결과**: 1-3 때보다 더 풍성하고 컨텍스트 있는 인사가 돌아옵니다. 회사·우선순위·톤이 다 반영된 첫 인사예요.
